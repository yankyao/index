# 2.常用布局

# 浮动的性质

  float属性，使用浮动可以让元素在一行排列，同时可以设置宽高。有两个属性值：left、right，在同一个盒子里，可以有不同方向的浮动。浮动使元素脱离标准流，变为块级元素，既可以设置宽高，也可以依次排列在一行。

  浮动方向分为左浮动和右浮动，多个元素在一个父盒子内，会依据标签书写的顺序依次向前贴边；如果父盒子的宽度不够装下元素盒子，则后面的盒子会自动换行。依次寻找上一个盒子，如果上一个盒子后面的宽度距离还不够，就找上上一个盒子，以此类推。子盒子的高矮不同，如果中间出现了缝隙，后面的盒子不会去钻空。浮动解决了margin塌陷现象。

  浮动元素让出标准流位置（产生压盖），元素在标准流文档中有自己的书写位置，一个元素写完之后才会加载另一个元素。设置浮动属性的元素和标准文档流出现在同一级父盒子里，会产生压盖现象。设置float属性的元素给标准文档流让位置，出现压盖在一起的效果(浮动的元素压盖不浮动的标准流元素)。在开发时，同一个父盒子内的元素，要么都设置浮动，要么都不设置浮动。

  浮动有字围效果，文字不像盒子背景部分，不会被浮动的元素遮盖住，文字会自动绕开浮动元素进行排列。

# 清除浮动

   浮动存在的问题，浮动的元素，不能撑高父盒子。元素脱离标准流之后，不会再撑高父盒子；浮动会影响后面的浮动的元素，两组浮动，如果浮动的方向相同，后一组浮动回去贴前一组浮动最后一个元素的边。

  解决方法：

  • 给父盒子添加高度，给父盒子增加相应的高度，其内部的浮动元素被限制了范围，则不会再影响后面的元素浮动。但是，仍然不能让父元素高度自适应。
  • 清除浮动属性clear，元素都可以设置一个clear属性，清除浮动的效果。clear有三个属性：left；right；both。给父盒子添加clear属性，可以使其内部的元素免受其他元素带来的浮动的影响。但是，父盒子的高度仍然不能自适应；且margin如果小于中间浮动的子元素的高度时，clear不能产生作用。
  • 隔墙法，外墙法：在父盒子中间加一个墙div{ clear:both;height:0px}模拟一堵墙，可以解决浮动影响；但，父盒子仍然不能自适应，且margin失效。在制作大的结构时，会在结构与结构之间加一堵外墙，分隔空间。内墙法：将清除浮动墙（带有clear属性和高度属性的div，属于标准流元素）放在浮动元素的父盒子内部元素的最后。可以解决高度自适应、浮动互相影响、margin失效的问题。缺点：①html结构布局，很多需要进行浮动，每个浮动的盒子内都需要隔一堵墙。页面结构复杂会出现很多冗余的没有意义的标签。一个盒子内的标签要浮动就都浮动，浮动后面的墙是一个标准流里的元素。
  • overflow，溢出模式属性，属性值为hidden时为溢出隐藏，属性值为auto时为溢出滚动（出现滚动条）。overflow属性可以清除盒子内部元素的浮动影响，完美解决了浮动互相影响、高度自适应、margin失效的问题，且保持结构的简洁，不会增加无用的标签。是工作中最常用的方式。没有设置高度，要使其自适应的盒子，设置overflow可以避免出错。overflow：hidden的加载机制：高度自适应：如果元素有溢出隐藏的属性，网页加载时，会强制的去元素内部搜索，识别所有内部元素的高度，会用最高的元素高度配给父盒子的自适应高度。浮动影响：加载时，如果有溢出隐藏，强制检索，看父盒子内部是否有浮动的元素，如果有，会被整体管理起来，在自身的高度内，不会影响其他的浮动元素。内部元素超出父盒子范围，超出父盒子的部分会自动隐藏。(隐藏在父盒子下面，但是，隐藏在父盒子padding下的部分，仍然会显示出来。) 整体布局时，合理设置overflow属性，避免出现不必要的滚动条。

# 定位

  css布局中：盒模型、浮动、定位，都是重要的布局属性。辅助布局属性：文本、a、背景等，是重要的样式属性。

  定位，position，是元素相对于某一参照物，进行的位置的偏移。(压盖效果；轮播图的图片切换，是JS调动定位)。定位的属性值分为：相对定位relative、绝对定位absolute、固定定位fixed、没有定位static、继承inherit。偏移的具体数据，由方向属性来控制，方向分为：left、right、top、bottom；属性值一般是像素，绝对定位可以是百分比（表示的是父盒子的百分比）。

{
    position：relative/absolute/fixed
    left/right：
    top/bottom：
}

  相对定位，相对定位指的是元素相对于自身的位置偏移；相对定位不会让元素脱离标准流，移动前的标准流的位置还是他自己的。定位的用途：通过相对定位进行小幅偏移，原来的位置保留，之前的结构不会被破坏，所以经常用作绝对定位的参考元素（子绝父相，设置相对定位属性，而不进行偏移。）；可以通过相对定位对元素的位置进行微调；可以做小特效。

  绝对定位，定位参考元素不固定，但是不可以参考自身。绝对定位使得元素脱离标准流，之前的标准留位置让给了后面的元素。参考元素为body时，top指的是首屏body页面的上角，bottom指的是首屏body页面的下角。参考元素为祖先元素时，元素绝对定位的参考元素是离他最近的有定位的祖先元素。（相对定位、绝对定位、固定定位都可以）。以父元素做为参考时，会忽视祖先元素的padding区域，其参考点为border内部的顶点做为参考方向；实际工作中，经常用子绝父相的方法，给父元素加相对定位而不进行偏移，结构稳定同时做为子元素的参考。绝对定位可以制作压盖效果。设置绝对定位元素居中的方式：水平居中：先移动父盒子的位置一半，再设置元素同方向的margin，使同方向margin的值为负的（子元素的一半的宽度）；同理，可以设置垂直居中；绝对定位的元素不会撑开父元素（做banner图的时候，常常用这个特点）。

  固定定位，固定定位参考的是浏览器窗口进行的定位，使元素的位置始终保持针对浏览器窗口某一顶点位置相对不变。

# z-index属性

  默认的压盖顺序：有定位（绝对定位或者固定定位）的元素压盖没有定位的元素；有定位（绝对定位或者固定定位）的元素不区分定位类型，按书写顺序，后面的压盖前面的。

  自定义压盖顺序：z-index，属性值为整数数字，数值大的压盖数值小的；数值相同看书写顺序，后面的压盖前面的；默认的压盖顺序中，实际上每个元素的z-index值为1；只能给定位元素加z-index，标准流元素和浮动元素都没有此属性；在父子盒模型中，如果父子都有定位，且都有z-index值，两对父子相比时，不论子元素的z-index值有多大，只要父盒子的index小，都只能被压盖。

# a标签

标签为页面上的所有链接规定默认地址或默认目标。假设图像在服务器文件夹中的绝对地址是：’/img’。在head部分可以插入base标签，规定页面所有的链接的基准URL。

<a href=”http://www.baidu.com” target=”_blank”>超级链接</a>

  a标签有四个状态，a:link，初始样式；a:visited，访问后显示的样式；a:hover，鼠标悬停的样式；a:active，鼠标点击不放显示的样式。四种显示状态的书写先后顺序不能颠倒，是a标签的四种伪类，在网页加载的时候不会加载，只有用户的行为触发了其中的状态，才会加载相应的样式。

  实际应用中，a标签是一个行内元素，不能设置宽高，所以很多时候，可以display：block(转块)；a标签的color、text-decoration不能继承父盒子，其他文本样式可以继承；实际工作中，默认样式都是四个状态设置成一样的（直接写在a标签内），需要哪个发生变化就单独设置这个伪类。a标签是行内元素，没脱标时，鼠标可以点击的区域仅仅是文字本身（font-size）的高度；脱标之后（display，浮动，定位），可以点击的是line-height撑高的区域；

  :hover，做为一个伪类，表示的是鼠标悬停在选择器所表示盒子范围内时，盒子的一些属性的变化，实质是鼠标悬停时，伪类变成真正的类，伪类中书写的属性及其值层叠掉了前面设置的（或默认设置的）一些属性及其值，所以伪类都写在后面才能层叠。伪类所选选择器的子盒子的属性同样可层叠（即分开写也属于同一事件）。可以制作很多小特效。

# 背景

  background-color背景色，属性值：十六进制、rgb、rgba、英文名。

  background-image背景图，实际渲染的位置和背景色一样，属性值url（uniform resource locator，统一资源定位符），可以是相对路径，也可以使绝对路径。默认状态下，背景图会铺满整个盒子；背景图压盖背景色；背景图不能撑开盒子，img插入图可以撑开盒子；背景图没有语义，img有语义相对重要的图片。

  background-repeat背景重复，属性值：repeat，默认值，整个背景区重复；no-repeat，不重复；repeat-x，水平方向重复；repeat-y，垂直方向重复。实际工作中，常用repeat-x制作渐变背景的导航栏。（切1像素宽、正常高度的渐变图片，水平方向重复）

  background-position背景图位置，设置插入的图片在盒子里的位置，属性值：像素表示、单词表示、百分比表示。第一个属性值为水平方向，第二个属性值为垂直方向，像素数值可以为负；设置一个合适盒子的大小，通过背景图的定位在网页中显示需要的图像（CSS精灵图技术）；单词，水平方向：left、center、right；垂直方向：top、center、bottom。单词一般用在body大背景图的设置上：水平居中，垂直居上(background-position:center top)；百分比，100%宽代表的像素=盒子背景宽度（盒子width+盒子border）-背景图的width。

  background-attachment背景附着，设置背景图片是否随着页面滚动而滚动。属性值：scroll、fixed。

复合书写，background：颜色 附着 url 重复 水平方向 垂直方向；

  背景的实际应用，在padding里面设置背景图。给盒子内部设置padding-方向，留出一个空白区域，然后根据空白区域大小，让图片以背景图方式插入空白区域并设置不重复。这样布局不会影响到实际可以书写内容的区域；图片替换文字，搜索引擎搜索时都是搜索文字，而不是图片。但是页面权重最大的logo部分一般显示为图片（h1一般放置logo）。h1中不插入图片，而是设置背景图；关键字可以大小设置为0或者缩进负的大像素，来隐藏文字。

h1 a {
    display: block;
    height: 96px;
    background: url( ) no-repeat;
    text-indent:-9999;
    overflow:hidden;
}

# 透明

  盒子透明，属性为opacity，取值范围0~1之间的小数。透明效果：盒子设置透明属性，内部元素也会透明。

  IE8及以下浏览器不支持opacity属性。只让背景透明，而内容不透明的方式：内容不放在背景透明的盒子，用定位定位到透明背景上面。IE8有自己的属性，filter属性：

filter: alpha(opacity=50)

# 浏览器兼容

  浏览器的份额，谷歌>ie8>ie9>ie7；谷歌内核webkit实际占有的市场份额将近60%；不同内核的浏览器对网页的兼容情况不同：一般来讲html网页的制作，兼容到ie8即可。

  HACK，针对不同的浏览器写的不同的html和css，以达到不同浏览器产生相同的渲染效果，称之为hack。hack分为html的hack和css的hack。

  html的hack是一组标签，做为可以规定特定版本的浏览器认识里面的代码的接口，可以放在html和css中；css的hack，分为选择器的hack、值的hack。值的hack（在属性前加）_下划线、-横线，只有IE6加载；~#@#$#@%#^等任一符号，只有IE6、7加载；在属性值后加\0/，只有IE8加载。选择器的hack，*html .selector{}，只有IE6及以下版本浏览器认识这个选择器；.selector，{}，只有IE7及以下版本浏览器认识这个选择器；html>body.selector{}，除了IE6版本，其他版本都认识这个浏览器；

<!--[if lte IE8]>
    这是一段代码或者标签或者文字或者图片
<![endif]—>
只能在IE8及以下浏览器中显示；
lte   less than or equal
lt    less than
gt    greater than
gte   greater than or equal
