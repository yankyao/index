# 2.html5

# 概述

2008 年发布。HTML5 不仅仅是超文本标记语言，而是一套完整的浏览器新 API 的综合；新的语义标签、新的本地存储、设备兼容特性（移动端）、连接特性、新的音频视频、新的画布和 svg、性能与集成特性、新的样式表；HTML5 不是 HTML 的新版本，而是包含 HTML、CSS、JavaScript、浏览器的一整套 API。

兼容情况：移动端兼容较好，PC端有一些兼容问题；API兼容性查询网站：html5test 和 can i use

新的骨架，新语义标签，新的语言规范；参考：MDN

# 新语义标签

HTML5 推出了新的大纲标签，他们都是双标签，都是div性质，块级元素，是容器标签。

<header>
  头部语义
  <footer>
    尾部语义
    <article>
      文章语义
      <section>
        区域语义
        <nav>
          导航语义
          <aside>
            侧边内容语义
            <main>主要内容语义</main>
          </aside>
        </nav>
      </section>
    </article>
  </footer>
</header>

# 新表单元素

以前type类型：button\checkbox\file\hidden\image\password\radio\reset\submit\text

新的type类型：number\date\week\color\time\range\search\tel

新的属性：required\autofouse\placeholder\readonly\pattern

<input
  type="number"
  min="2"
  max="140"
  placeholder="请输入2-140之间的数字"
/>
<input type='text' required pattern="^[\d]{2,5}$"/>

新的  控件，label 简化写法，直接包裹即可

<input type="text" list="tip" />
<datalist id="tip">
  <option value="HTML">超文本标记语言</option>
  <option value="JS">JS脚本语言</option>
  <option value="CSS">层叠式样式表</option>
</datalist>
<button>按钮</button>
<label><input type="checkbox" />足球</label>

新的表单事件：oninput，实时监听文本框输入。

相比使用onchange/onkeyup/onkeypress/onkeydown组合实现：

  • onchange 事件只在键盘或者鼠标操作改变对象属性，且失去焦点时触发，脚本触发无效；
  • onkeydown/onkeypress/onkeyup 在处理复制、粘贴、拖拽、长按键（按住键盘不放）等细节上并不完善。

onpropertychange IE9以下不支持 oninput。需要综合 oninput/onpropertychange 二者来实现文本区域实时监听的功能。

# 新的全局属性

contenteditable，模拟输入框。但是复制到模拟输入框元素的文本会源保留格式，input 则不会保留格式。

<!-- 使用div模拟input -->
<style>
  .input {
    width: 200px;
    height: 24px;
    line-height: 24px;
    font-size: 14px;
    padding: 5px 8px;
    border: 1px solid #ddd;
    border-radius: 12px;
  }
  .input:empty::before {
    /*  模拟 placeholder attr(placeholder)定义显示在该选择器之前或之后的选择器的选择的dom元素的某属性。 */
    content: attr(placeholder);
  }
</style>

<body>
  <div class="input" contenteditable placeholder="请输入文字"></div>
  <script>
    var input = document.getElementsByClassName("input")[0];
    input.oninput = function(event) {
      console.log(event.target.innerHTML);
    };
  </script>
</body>

# 视频和音频

不用 H5 插入视频的步骤：将视频上传到视频网站上（优酷、土豆等等），这样就不用买视频服务器。通过分享得到引用链接，使用 Flash 播放。

<embed src='https://player.youku.com/…/…**.swf' quality='high' width='100px' height='100px' allowScriptAccess='always' allowFullScreen='true' mode='transparent' type='application/x-shockwave-flash' />

H5 中引入了 video 标签，src 资源地址，autoplay 自动播放，controls 显示控制按钮，loop 循环播放。视频的方法有 pause()、play()：

<video src='video/lcw.mp4' width='100px' height='100px' autoplay controls loop></video>

音频 audio 标签，使用方法与 video 相同。

# 本地存储

Cookie 与 LocalStorage、SessionStorage，都是用户浏览器的本地存储。 cookie：

  • 服务器在http响应头中Set-Cookie: cookie，在用户浏览器中存储Cookie字符串；
  • 使用document.cookie来获取和修改cookie；
  • 同源http请求会自动携带cookie在请求头上；

H5新API可以在客户端的电脑中存储更多数据(5m)，LocalStorage（本地存储）、SessionStorage（会话存储），统称为本地存储。

本地存储数据库会自动的为每一个网站（ip 地址）建立一个独立的表格，在同一个网站（ip 地址）下数据是可以共享的。

  • localStorage存储的东西，是没有过期时间。
  • sessionStorage 就不一样，它是浏览器窗口一旦关闭，数据丢失了。
  • localStorage 不能跨浏览器存储。
  • 存储形式是kv对，v数据的通用形式是字符串。存储JSON需要，JSON.stringify();
localStorage.setItem(k, v);
localStorage.getItem(k);
localStorage.removeItem(k);
localStorage.clear();

# websocket

WebSocket protocol 是 HTML5 一种新的协议。它实现了浏览器与服务器全双工通信(full-duple)。一开始的握手需要借助 HTTP 请求完成。

  • HTTP 设计的时候，比如浏览器直接发出请求，服务器才能响应。如果浏览器不发出 request 请求，服务器是不能主动找到浏览器，传输一些数据的。也就是说：浏览器必须主动请求，服务器才会发出响应。

但是现在的 web 页面，对实时的要求很高：web 看股票、篮球比赛图文直播、聊天室、站内信等等。现在的做法基本都是长轮询，用通俗易懂的话来说，就是客户端定时（每隔大约 1 秒钟）向服务器发送请求以获取最新的数据信息。断开、连接、断开、连接。服务器是不能有新消息就主动通知浏览器的。

Header，互相沟通的Header是很小的-大概只有 2 Bytes；
Server Push，服务器的推送，服务器不再被动的接收到浏览器的request之后才返回数据，而是在有新数据时就主动推送给浏览器。

websocket 需要浏览器足够新，IE10+。服务器也要足够新，NodeJS 天生就支持。

socket.io 可以提高开发 websocket 的效率，已经封装好了诸如 HTTP 头等

# 应用缓存 Manifest

  • 使用Manifest文件可以创建缓存清单；
  • 网络无连接应用仍可用这些缓存；
  • 本地读取缓存资源，提升访问速度，增强用户体验；
  • 减少请求，缓解服务器负担。
// manifest.json
CACHE MANIFEST

#要缓存的文件
CACHE:
    images/img1.jpg
    images/img2.jpg


#指定必须联网才能访问的文件
NETWORK:
     images/img3.jpg
     images/img4.jpg


#当前页面无法访问是回退的页面
FALLBACK:
    404.html

使用：

<!DOCTYPE html>
<html manifest="manifest.json">
<head lang="en">
    <meta charset="UTF-8">
    <title></title>
</head>
<body>
<img src="http://img.smyhvae.com/2016040101.jpg" alt=""/>
</body>
</html>

# 多任务 Webworker

Web Worker 的作用，就是为 JavaScript 创造多线程环境，允许主线程创建 Worker 线程，将一些任务分配给后者运行。 注意点：

  • 同源限制；
  • DOM限制；
  • 通信联系：通过消息完成worker.postMessage；
  • 脚本限制：不能执行alert和confirm方法，但可以使用 XMLHttpRequest 对象发出AJAX请求。
  • 文件限制：不能打开本机的文件系统，加载的脚本必须来自网络。

# 跨窗口通信 PostMessage

H5中新增的postMessage()方法，可以用来做跨域通信。一般是搭配iframe来实现跨域：

场景：窗口 A (http:A.com)向跨域的内嵌iframe窗口B (http:B.com)发送信息。步骤如下。

（1）在A窗口中操作如下：向B窗口发送数据：

 	Bwindow.postMessage('data', 'http://B.com'); //这里强调的是B窗口里的window对象

（2）在B窗口中操作如下：

    // 在窗口B中监听 message 事件
    Awindow.addEventListener('message', function (event) {   //这里强调的是A窗口里的window对象
        console.log(event.origin);  //获取 ：url。这里指：http://A.com
        console.log(event.source);  //获取：A window对象
        console.log(event.data);    //获取传过来的数据
    }, false);

# 拖拽释放(Drag and drop) API

通过为元素增加 draggable="true" 属性，来设置此元素是否可以进行拖拽操作，其中图片、链接默认是开启拖拽的。

有自己的事件：

  • ondragstart当拖拽开始时调用
  • ondragleave 当鼠标离开拖拽元素时调用
  • ondragend 当拖拽结束时调用
  • ondrag 整个拖拽过程都会调用
