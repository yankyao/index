# 1.css基础

# 概念

cascading style sheet，层叠式样式表。规定了html标签在网页上的显示样式。html4时，w3c组织将html的结构和样式进行了分离。前端分为三个层次：html 结构层，搭建了网页的整体架构；css 样式层，装饰了页面；JS 行为层，页面交互效果的实现。

css有两个重要的概念：层叠式和样式。样式指的是html在页面的显示效果；层叠式，指的是显示哪些效果，覆盖掉哪些效果。通过css设置样式，有两个关键：选择器和样式表。

css的样式表可以分为两部分：

  1. 给文本添加文字显示样式；
  2. 给标签盒子添加属性进行结构布局。

主要关键词：文本、盒模型、背景、浮动、定位、flex、grid。

  • html的属性和属性值，键值对：k="v"；css的属性和属性值，键值对：k:v;
  • （除了字体font-family），多个属性之间必须用分号分割，不写出错。
  • 内嵌式：所有的样式必须写在一对大括号里。
  • css样式，对于换行、空格、缩进也不敏感。为了提高加载速度，需要将所有的空格、缩进、换行等删除，压缩代码。

# 常用属性

  1. 文本样式：字体font-family、颜色color、大小font-size。字体要用引号包裹，字体之间用逗号隔开，表示选择，为了追求加载速度，一般把字体中文名写成英文名。书写的时候将英文字体写在前面，中文字体在后面。微软雅黑：Microsoft Yahei；隶书：SimSun；颜色有多种表示方法，16进制、RGB、rgba、英文颜色名。

  2. 盒子实体化基本属性：将盒子实体化，width、height、background、border、position、display、overflow、margin、padding等。

# css的书写位置

  • 行内式；
  • 内嵌式；
  • 外链式；
<!-- 行内式 -->
<div style="color:red;font-size:30px;width:200px;">
    这是一个div
</div> 
<!-- 内嵌式,写在head部分，style标签包裹 -->
<style type="text/css">
    .divide {
        display: none;
    }
</style> 
<!-- 外链式, 将css单独写在另外一个.css文件里 -->
<link rel="stylesheet" type="text/css" href="demo.css"></link>

TIP

优先级：!important> 行内式 > 内嵌式 = 外链式。

  • 继承性：后代元素能够继承来自祖先元素的文字的样式。不能继承盒子的样式。继承性这个特性，我们可以将整体风格的文字样式写在最祖先元素body，后面的标签自动去继承。有一些标签的一些属性不自动继承祖先元素，比如a标签的text-decroation、color，h标签的font-size等等。

  • 层叠性：都没有选中标签，则考虑继承性，继承性第一原则是就近原则；同样近的则看各个样式的权重。权重的比较原则：都选中元素的情况下，先比较ID选择器个数，再比较类选择器个数，再比较标签选择器个数；都相同的看书写顺序，则是后写的样式。

# 基础选择器

  • 标签选择器：通过标签名直接选择相应的标签。标签是什么我们就用什么选择。标签选择器选中的是页面所有的对应标签。因为标签选择器会选中所有的标签，实际工作中，不会用来更改某一个元素属性，因为会影响其他。用途：利用标签选择所有，进行初始样式的设置，默认样式的清除。

  • ID选择器：通过标签的id属性值来选择相应的标签。#开头，后面紧跟id名，中间没有任何空格。id选择器只能选中一个元素，一个页面不允许出现两个相同的id名，即使是不同类型的标签。id名命名必须以字母开头（严格区分大小写，A和a表示的不一样），后面可以是字母、数字、下划线或者横线。用途：经常不是用来添加样式，而是留给js添加行为。

  • 类选择器（常用）：通过标签的class属性来选择这个标签，选择器写法：.开头，后面class属性值，中间不能有空格。class属性，值可以不唯一。类选择器选中的是所有相同的class名的元素。标签的class属性，属性值命名规则与id一样。优点：可以选择一部分标签，添加相同的样式。根据类的特性，我们要习惯使用原子类（通用类）。原子类设置的原则，里面只设置一个单一的css属性，某一个标签需要添加这个属性，可以直接添加这个原子类的类名。

  • 通配符*：可以选中包括body在内的所有标签，主要用来清空默认样式。实际工作中清默认样式：用的是有默认样式的标签选择器。

  • 其他选择器：属性选择器等；

span[target="1"] { /* 元素target属性的值为1 */
    color: red
}
span[target~="1"] { /* target的值包含1 */
    color: red
}
span[target|="1"] { /* target的值以1开头 */
    color: red
}
div > p {} /* 选择父元素为 <div> 元素的所有 <p> 元素 */
div + p {} /* 选择紧接在 <div> 元素之后的所有 <p> 元素 */

a:link {} /* 选择所有未被访问的链接 */
a:visited {} /* 选择所有已被访问的链接 */
a:active {} /* 选择活动链接 */
a:hover {} /* 选择鼠标指针位于其上的链接 */

input:focus {} /* 选择获得焦点的 input 元素 */

p:first-letter {} /* p元素的首字母 */
p:first-line {} /* p元素的首行 */
p:first-child {} /* 属于父元素的第一个子p元素 */

# 高级选择器

三种高级选择器是基础选择器组合用法。

  • 后代选择器：通过标签之间的嵌套关系、层级关系，限定我们大体范围，在范围内具体查找相关元素。根据嵌套关系，空格左侧是右侧的祖先元素。后代选择器空格两边只要是后代关系都可以，不一定是父子关系。一般的开始的祖先元素都不使用标签选择器。

  • 交集选择器：两种或者以上基础选择器同时存在一个标签上。选择器之间直接连接，没有任何符号（没有空格）。

  • 并集选择器：有很多元素可能都有相同css样式，我们可以通过并集选择器将这些标签写在一起，统一设置css样式。前面所有的选择器都可以作为并集里的一项，在选择器中间用逗号隔开。

# 文字样式属性

文字样式包括字体样式和文本样式:

字体样式：颜色、字体、字号、行高、字体粗细、字体的一些样式。

font: font-weight font-style font-size line-height font-family;/* font里面最好不要合写进color，不同版本有的不支持，最好单独设置； */
color: 16进制、RGB、rgba、颜色名
font-family，中文一般有微软雅黑、宋体；英文有Arial、Consolas。先规定英文字体，再规定中文字体，整体要用双引号引住，先后顺序为''或"的选择关系。
font-size，常用12/14/16/18px，可用fireworks进行比对测量，注意要用两个字或以上进行比对（不消除锯齿），完全重合则是对的。(chrome最小支持12px)
line-height，单位为像素或则百分比，一行文字实际占有的高度，行高中的文字是垂直居中的。可以量两行文字底边的距离。行高属性在书写时，必须写在文字字号的后面。要让一行文字在盒子里垂直居中（如新闻标题），让行高等于盒子高。
font-style，字体样式。normal正常，italic斜体（不常用），oblique斜体（常用）
font-weight，字体粗细。normal正常，bold加粗。

文本样式，对齐、文本缩进、文本修饰。

text-align，文字对齐。左对齐、右对齐、居中。两端对齐，justify;(会受padding影响)
text-indent，文字首行缩进。单位em为两个汉字宽度；百分比则按父盒子的比例计算；也可以用像素。
text-decoration，文字修饰。正常，没有修饰：none；下划线：underline；中划线，删除线：line-through；上划线：overline。
所有的文本（除了a标签内），默认属性值都是none。

# 盒模型

盒模型也叫框模型，包含五部分：宽度、高度、内边距、边框、外边距。单位都可以是px % em rem vw vh等。

  1. box-sizing，盒子实际占有的区域：

  • content-box: 宽=width+（padding-right+padding-left）+（border-right+border-left）; 高=height+（padding-top+padding-bottom）+（border-top+padding-bottom）。

    要增加内边距和边框，则需要对宽高进行内减。

  • border-box：宽 = width;高 = height

  1. padding，内边距。四值：上、右、下、左；三值：上、左右、下；二值：上下、左右。

  2. border，边框。也分为四部分。写法与padding一样。

  • border-color写法与padding一样。

  • border-style有border-top/right/bottom/left-style设置四条边的属性，常用属性值：solid 实线；dashed 虚线;dotted 点的虚线(宽度默认三像素，需要设置);double 双线条;groove 边框凹陷效果;ridge 边框凸出效果;inset 内容区域凹陷效果;outset 内容区域凸出效果

  • 单边属性写法：border-top: 5px solid red；

  • border-collapse，table标签专用：边框合并属性。默认值：separate，合并值：collapse。

  1. margin，外边距：外边距是盒子与盒子之间的距离，设置方法类似padding。盒子可以在页面水平居中（margin：auto）

TIP

小技巧：margin-left:-1px; 设置margin-left避免用覆盖清除边框时，拐角处出现边框缺失

# 盒模型拓展

  • 清空默认样式，很多标签都有一些默认的盒子样式，body、p、div、ul、li、ol、dl、dt、dd、h系列等都有默认的内外边距；ul、ol都有默认的list-style列表默认样式，a有默认的字体样式，包括下划线、颜色和点击颜色变化。这些在开发的第一步，都需要预先清除。

  • 盒子的height（高度剩余），一般不设置盒子的height或者设置为auto，而是用内部的元素来撑高。是为了保证我们在删除或者添加的时候，网页的高度可可以自适应内容而变化。是因为网页的结构和样式是前端开发，而具体填充的内容是后台数据导入的，需要灵活性。盒子内元素溢出：设置overflow可以控制溢出后的样式。 宽度剩余，与高度剩余类似。

  • 外边距塌陷(margin塌陷)，在垂直方向上，相邻的两个元素如果都有外边距，而且外边距有相交的部分，相交的外边距会出现重合。边距显示为边距较大的值。嵌套关系的盒子，同理。（没有设置外边距，实际上外边距的值为0px）。解决方法：实际工作中，垂直方向上的两个相邻盒子，要么给上边的盒子设置下边距，要么给下边的盒子设置上边距，不要同时设置；嵌套关系的两个盒子强行给父盒子加上一个边框，强制限定父盒子的位置，或者不设置子盒子的margin，设置父盒子的padding，达到想要的效果。（常用）

  • 居中，文字水平方向上的居中：text-align：center；垂直方向上的居中：单行文字，让行高（line-height）= 盒子高;盒子居中：margin：0 auto；文本、子盒子在盒子内居中：不给盒子设置高度，给盒子设置上下的padding，盒子的高度会自适应。

  • 父子盒模型，父盒子的区域的宽度≥子盒子的width+paddingx2+borderx2;如果子盒子不设置宽度，其中的子元素可以撑满父盒子，宽度是父盒子的100%（区别于继承性）。如果你再设置了子盒子的边框和内边距，自动向内减少，不需要你手动的去减少父盒子的width。

.outer {
    width: 500px;
    border: 1px solid #000;
    margin: 100px 200px;
 }
.inner {
    padding: 10px 20px;
    background: skyblue;
    line-height: 28px;
    border: 10px solid red;
}
/* 其中，父盒子的总体宽度为502px； */

# 标准文档流

html网页是标准文档流，指的是我们的书写内容是从上到下、从左到右的渲染顺序。前面元素的大小或者内容性质发生变化，后面的内容也会发生一些位置变化。从微观上有的特殊性质：html文本之间有空白折叠现象；文字、图片、表单等文本类型的元素，有一个特点：高矮不齐、底边对齐；文字在盒子内填充完一行，会自动换行。

块级元素和行内元素，标准文档流中的标签元素，可以分为块级标签、行内标签、行内块标签。

  • 块级标签：所有的容器级标签都是块级标签，文本级中的P标签也是一个块级标签。如：body，div，h1，P，ul，ol，li，dl，dt，table，tr等等；块级标签元素会独占一行，且同行内不会出现其他的同级标签；块级标签可以设置宽高，在嵌套关系中，子盒子设置了高度不设置宽度，会自动继承父盒子宽度的100%。

  • 行内标签：除了P标签以外的所有文本级标签，如：a、span、img、input等。行内标签不独占一行，可以与其他的行内标签元素并排一行。行内元素不能设置宽高，其他的外边距( 但是不能margin：0 auto; )、边框、内边距可以正常设置；行内元素不设置宽度，不会自动继承父盒子的宽度，用内容撑开。

  • 行内块元素：既可以设置宽高，也可以并排显示。

/* 设置显示模式属性：display。 */
{
    display：none; /* 不显示这个元素； */
    display：inline; /* 显示为行内元素；（不常用） */
    display：block; /* 显示为块级元素，可以设置宽高，但是未脱标；（常用） */
    display: inline-block; /* 在文字行高撑起来的盒子里插入背景时，会用到； */
}

标准文档流制作网页有很多局限，很多布局效果不能实现。让元素脱离标准流，既可以设置宽高，又可以在一行排列的方法有三种：浮动、绝对定位、固定定位、设置display属性。有四者之一，即可设置宽高。

# 层叠性-权重

  1. 内联样式，style=""，权值为 1000
  2. id选择器，如#content，权值为100
  3. 类，伪类和属性选择器，如.content，权值为10
  4. 标签选择器和伪元素选择器，如div p，权值为1

TIP

通用选择器（*），子选择器（>），和相邻同胞选择器（+）权值为0

# BFC

Block formatting context，块级格式化上下文。一个隔离的独立容器，容器里面的子元素不会影响到外面的元素。 表现：

  • 内部的Box会在垂直方向，一个接一个地放置。
  • Box垂直方向的距离由margin决定。属于同一个BFC的两个相邻Box的margin会发生重叠。

创建BFC：

  • float的值不是none。
  • position的值不是static或者relative。
  • display的值是inline-block、table-cell、flex、table-caption或者inline-flex；
  • overflow的值不是visible；

# BEM

一种CSS Class 命名方法，旨在更好的创建CSS/Sass模块。

# 预处理器

  1. scss
  2. less
  3. stylus

TIP

面试常问：scss和less的区别：

  1. less是通过js编译是在客户端处理；sass同通过ruby是在服务器端（自己开了个node进程，node-sass）处理；
  2. 变量符号：less是用@,sass是用$；
  3. sass支持条件语句，可以使用if{}else{},for{}循环等等。而less不支持。
  4. less没有输出设置，sass提供4中输出选项：nested, compact, compressed 和 expanded。
