# 2.常用布局

# 标准流文档

元素在标准流文档中有自己的位置，一个元素写完之后才会加载另一个元素。

# 浮动的性质

float属性：

  • 使元素脱离标准流，变为块级元素，既可以设置宽高，也可以依次排列在一行。
  • 使用浮动可以让元素在一行排列，同时可以设置宽高。有两个属性值：left、right。父元素的宽度不够装下元素元素，则后面的元素会自动换行。
  • 子元素的高矮不同，如果中间出现了缝隙，后面的元素不会去钻空。浮动解决了margin塌陷现象。
  • 在同一个元素里，可以有不同方向的浮动。
  • 浮动有字围效果，文字不像元素背景部分，不会被浮动的元素遮盖住，文字会自动绕开浮动元素进行排列。

# 清除浮动

浮动存在的问题：

  • 浮动的元素，不能撑高父元素。元素脱离标准流之后，不会再撑高父元素；
  • 浮动会影响后面的浮动的元素，两组浮动，如果浮动的方向相同，后一组浮动回去贴前一组浮动最后一个元素的边。

解决方法：

  • 给父元素添加高度，给父元素增加相应的高度，其内部的浮动元素被限制了范围，则不会再影响后面的元素浮动。但是，仍然不能让父元素高度自适应。

  • 清除浮动属性clear，元素都可以设置一个clear属性，清除浮动的效果。clear有三个属性：left；right；both。给父元素添加clear属性，可以使其内部的元素免受其他元素带来的浮动的影响。但是，父元素的高度仍然不能自适应；且margin如果小于中间浮动的子元素的高度时，clear不能产生作用。

  • 隔墙法，外墙法：在父元素中间加一个墙div{ clear:both;height:0px}模拟一堵墙，可以解决浮动影响；但，父元素仍然不能自适应，且margin失效。在制作大的结构时，会在结构与结构之间加一堵外墙，分隔空间。内墙法：将清除浮动墙（带有clear属性和高度属性的div，属于标准流元素）放在浮动元素的父元素内部元素的最后。可以解决高度自适应、浮动互相影响、margin失效的问题。缺点：①html结构布局，很多需要进行浮动，每个浮动的元素内都需要隔一堵墙。页面结构复杂会出现很多冗余的没有意义的标签。一个元素内的标签要浮动就都浮动，浮动后面的墙是一个标准流里的元素。

  • overflow，溢出模式属性，属性值为hidden时为溢出隐藏，属性值为auto时为溢出滚动（出现滚动条）。 overflow属性可以清除元素内部元素的浮动影响，完美解决了浮动互相影响、高度自适应、margin失效的问题，且保持结构的简洁，不会增加无用的标签。是工作中最常用的方式。没有设置高度，要使其自适应的元素，设置overflow可以避免出错。overflow：hidden的加载机制：高度自适应：如果元素有溢出隐藏的属性，网页加载时，会强制的去元素内部搜索，识别所有内部元素的高度，会用最高的元素高度配给父元素的自适应高度。浮动影响：加载时，如果有溢出隐藏，强制检索，看父元素内部是否有浮动的元素，如果有，会被整体管理起来，在自身的高度内，不会影响其他的浮动元素。内部元素超出父元素范围，超出父元素的部分会自动隐藏。(隐藏在父元素下面，但是，隐藏在父元素padding下的部分，仍然会显示出来。) 整体布局时，合理设置overflow属性，避免出现不必要的滚动条。

# 定位

position，是元素相对于某一参照物，进行的位置的偏移。(压盖效果；轮播图的图片切换，是JS调动定位)。

  • 定位的属性值分为：相对定位relative、绝对定位absolute、固定定位fixed、没有定位static、继承inherit、粘性sticky。
  • 偏移的具体数据，由方向属性来控制，方向分为：left、right、top、bottom；属性值一般是像素，绝对定位可以是百分比（表示的是父元素的百分比）。

left/right：
top/bottom：

  • 相对定位，相对定位指的是元素相对于自身的位置偏移；相对定位不会让元素脱离标准流，移动前的标准流的位置还是他自己的。定位的用途：通过相对定位进行小幅偏移，原来的位置保留，之前的结构不会被破坏，所以经常用作绝对定位的参考元素（子绝父相，设置相对定位属性，而不进行偏移。）；可以通过相对定位对元素的位置进行微调；可以做小特效。

  • 绝对定位，定位参考元素不固定，但是不可以参考自身。绝对定位使得元素脱离标准流，之前的标准留位置让给了后面的元素。参考元素为body时，top指的是首屏body页面的上角，bottom指的是首屏body页面的下角。参考元素为祖先元素时，元素绝对定位的参考元素是离他最近的有定位的祖先元素。（相对定位、绝对定位、固定定位都可以）。以父元素做为参考时，会忽视祖先元素的padding区域，其参考点为border内部的顶点做为参考方向；实际工作中，经常用子绝父相的方法，给父元素加相对定位而不进行偏移，结构稳定同时做为子元素的参考。绝对定位可以制作压盖效果。设置绝对定位元素居中的方式：水平居中：先移动父元素的位置一半，再设置元素同方向的margin，使同方向margin的值为负的（子元素的一半的宽度）；同理，可以设置垂直居中；绝对定位的元素不会撑开父元素（做banner图的时候，常常用这个特点）。

  • 固定定位，固定定位参考的是浏览器窗口进行的定位，使元素的位置始终保持针对浏览器窗口某一顶点位置相对不变。

  • sticky，监听页面 scroll 事件，判断每一区块距离视口顶部距离，超过了则设定该区块 position:fixed，反之去掉。

# z-index属性

默认的压盖顺序：

  • 有定位（绝对定位或者固定定位）的元素压盖没有定位的元素；
  • 有定位（绝对定位或者固定定位）的元素不区分定位类型，按书写顺序，后面的压盖前面的。

自定义压盖顺序：z-index，属性值为整数数字：

  • 数值大的压盖数值小的；
  • 数值相同看书写顺序，后面的压盖前面的；默认的压盖顺序中；
  • 在父子盒模型中，如果父子都有定位，且都有z-index值，两对父子相比时，不论子元素的z-index值有多大，只要父元素的index小，都只能被压盖。

每个元素的z-index值默认值为1。只能给定位元素加z-index，标准流元素和浮动元素都没有此属性。

# a标签

base标签为页面上的所有链接规定默认地址或默认目标。

head部分可以插入base标签，规定页面所有的链接的基准URL。

<a href="http://www.baidu.com" target="_blank">超级链接</a>

a标签有四个状态:

  • a:link，初始样式；
  • a:visited，访问后显示的样式；
  • a:hover，鼠标悬停的样式；
  • a:active，鼠标点击不放显示的样式。

默认样式都是四个状态设置成一样的（直接写在a标签内），需要哪个发生变化就单独设置这个伪类。

四种显示状态的书写先后顺序不能颠倒，是a标签的四种伪类，在网页加载的时候不会加载，只有用户的行为触发了其中的状态，才会加载相应的样式。

实际应用中，a标签是一个行内元素，不能设置宽高，所以很多时候，可以display：block(转块)；

a标签的color、text-decoration不能继承父元素，其他文本样式可以继承；

a标签是行内元素，没脱标时，鼠标可以点击的区域仅仅是文字本身（font-size）的高度；

# 背景

  • background-color背景色，属性值：十六进制、rgb、rgba、英文名。

  • background-image背景图，实际渲染的位置和背景色一样，属性值url（uniform resource locator，统一资源定位符），可以是相对路径，也可以使绝对路径。默认状态下，背景图会铺满整个元素；背景图压盖背景色；背景图不能撑开元素，img插入图可以撑开元素；背景图没有语义，img有语义相对重要的图片。

  • background-repeat背景重复，属性值：repeat，默认值，整个背景区重复；no-repeat，不重复；repeat-x，水平方向重复；repeat-y，垂直方向重复。实际工作中，常用repeat-x制作渐变背景的导航栏。（切1像素宽、正常高度的渐变图片，水平方向重复）

  • background-position背景图位置，设置插入的图片在元素里的位置，属性值：像素表示、单词表示、百分比表示。第一个属性值为水平方向，第二个属性值为垂直方向，像素数值可以为负；设置一个合适元素的大小，通过背景图的定位在网页中显示需要的图像（CSS精灵图技术）；单词，水平方向：left、center、right；垂直方向：top、center、bottom。单词一般用在body大背景图的设置上：水平居中，垂直居上(background-position:center top)；百分比，100%宽代表的像素=元素背景宽度（元素width+元素border）-背景图的width。

  • background-attachment背景附着，设置背景图片是否随着页面滚动而滚动。属性值：scroll、fixed。

复合书写，background：颜色 附着 url 重复 水平方向 垂直方向；

背景的实际应用，在padding里面设置背景图。给元素内部设置padding-方向，留出一个空白区域，然后根据空白区域大小，让图片以背景图方式插入空白区域并设置不重复。这样布局不会影响到实际可以书写内容的区域；图片替换文字，搜索引擎搜索时都是搜索文字，而不是图片。**但是页面权重最大的logo部分一般显示为图片（h1一般放置logo，为了seo）。**h1中不插入图片，而是设置背景图；关键字可以大小设置为0或者缩进负的大像素，来隐藏文字。

h1 a {
    display: block;
    height: 96px;
    background: url( ) no-repeat;
    text-indent:-9999;
    overflow:hidden;
}

# 透明

元素透明，属性为opacity，取值范围0~1之间的小数。透明效果：元素设置透明属性，内部元素也会透明。

IE8及以下浏览器不支持opacity属性。只让背景透明，而内容不透明的方式：内容不放在背景透明的元素，用定位定位到透明背景上面。IE8有自己的属性，filter属性：

filter: alpha(opacity=50)

# 浏览器兼容

谷歌内核webkit实际占有的市场份额将近60%；不同内核的浏览器对网页的兼容情况不同。

HACK，针对不同的浏览器写的不同的html和css，以达到不同浏览器产生相同的渲染效果，称之为hack。hack分为html的hack和css的hack。

html的hack是一组标签，做为可以规定特定版本的浏览器认识里面的代码的接口，可以放在html和css中；css的hack，分为选择器的hack、值的hack。

值的hack（在属性前加）:

  • _下划线、-横线，只有IE6加载；
  • ~#@#$#@%#^等任一符号，只有IE6、7加载；
  • 在属性值后加\0/，只有IE8加载。

选择器的hack:

  • *html .selector{}，只有IE6及以下版本浏览器认识这个选择器；
  • .selector，{}，只有IE7及以下版本浏览器认识这个选择器；
  • html>body.selector{}，除了IE6版本，其他版本都认识这个浏览器；
<!--[if lte IE8]>
    这是一段代码或者标签或者文字或者图片
<![endif]—>
只能在IE8及以下浏览器中显示；
lte   less than or equal
lt    less than
gt    greater than
gte   greater than or equal

# Bootstrap

BootStrap是一个前端开发框架；适用于敏捷开发。Bootstrap，来自Twitter，是目前很受欢迎的前端框架。Bootstrap是基于HTML、CSS、JAVASCRIPT的，它简洁灵活，使得Web开发更加快捷。。

现在的版本是2、3、4代：

  • 2、3都是底层预处理css语言都是less;
  • 3代开始扁平化风格UI；
  • 4代开始sass处理语言；
