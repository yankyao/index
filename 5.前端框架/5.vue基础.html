# 5.vue基础

# 简介

Vue和React有许多相似之处，它们都有：使用VirtualDOM，提供了响应式（Reactive，数据驱动视图）和组件化（Composable）的视图组件。将注意力集中保持在核心库，专注于UI层，伴随于此有配套的路由和负责处理全局状态管理的JS库(全家桶)。

  • React主流使用jsx语法糖，JSX的渲染功能有下面这些优势：可以使用完整的JS功能写组件，使用JS灵活度高；工具对JSX的支持比Vue模板更完善（比如，linting、类型检查、编辑器的自动完成）。

  • Vue主流使用模板语法：在写模板的过程中，样式风格已定并涉及更少的功能实现；模板中任何 HTML 语法都是有效的；阅读起来更贴合英语（比如，for each item in items）；不需要高级版本的 JavaScript 语法，来增加可读性。

Vue.js的核心是一个响应的数据绑定系统，拥抱数据驱动的视图概念。每当修改了数据，DOM便相应地更新。应用中的逻辑就都是修改数据来改变视图，不必与DOM更新搅在一起。

让代码更容易开发、理解、维护。

// 两个大括号表示插入数据
// 三个大括号则可以识别html标签
// 注意与Jsx的区别
<p>{{text}}</p> // <span>你好</span>
<p>{{{text}}}</p> // 你好

data: {
    text: '<span>你好</span>'
}

# 基本的工作机制

在new Vue()之后，Vue会调用初始化函数，会初始化声明周期、事件、props、methods、data、computed和watcher等。其中最重要的是通过Object.defineProperty设置setter和getter，用来实现响应式和依赖收集。初始化之后会调用$.mount挂载组件。

  • parse，使用正则解析模板中的vue的指令、变量等等，形成抽象语法树AST；
  • optimize，标记一些静态节点，用作后面的性能优化，在diff的时候直接略过；然后递归做patch;
  • generate，把第一步生成的AST转化为渲染函数 render函数；

响应式是vue最核心的内容。初始化的时候通过defineProperty进行绑定，使用订阅发布的设计模式：当编译生成的渲染函数被实际渲染的时候，会触发getter进行依赖收集，在数据变化的时候，触发setter进行更新。

  • 虚拟dom是由react首创，Vue2开始支持，就是用JavaScript对象来描述dom结构，数据修改的时候，先修改虚拟dom中的数据，然后vm数组做diff算法，最后再汇总所有的diff，力求做最少的dom操作，毕竟js里对比很快，而真实的dom操作比较慢。

# Demo

官方tudolist例子,可以添加事件和删除todo事件：

<div id="box">
    <input type="text" v-model='txt' @keyup.enter='addItem'>
    {{txt}}
    <ul>
        <li v-for='(item, i) in list'>{{item}}<button @click='deleteItem(i)'>删除</button></li>
    </ul>
</div>
<script src='./node_modules/vue/dist/vue.min.js'></script>
<script>
    new Vue({
        el: '#box',
        data: {
            txt: '',
            list: []
        },
        methods: {
            // 此处不能使用剪头函数，此处如果定义了箭头函数的this指向window。
            // 使用functino则指向调用者，即Vue实例。
            // 要注意与Vue工程化组件时的区别
            addItem() {
                if(this.txt) {
                    this.list.push(this.txt)
                    this.txt = ''
                }
            },
            // 此处，不能命名为delete，因为delete为关键字
            deleteItem(index) {
                this.list.splice(index, 1)
            }
        }
    })
</script>

# 实例

一个Vue实例其实正是一个MVVM模式中所描述的ViewModel因此在文档中经常会使用vm这个变量名。 data属性是一个对象，这个JSON中的所有属性就是实例的属性。每一个vue实例都可以直接用点语法得到自己的data中的属性。还有$data、$el、$watch内置属性或方法。

<script>
    var vm = new Vue({
        el: '#box',
        data: {
            number: 888,
        },
        methods: {
            plus() {
                this.number++
            }
        },
        watch:{
            number: function(newValue, oldValue) {
                console.log(newValue, oldValue)
            }
        }
    })
    console.log(vm.number) // 888
    console.log(vm.$data.number) // 888
    console.log(vm.$el)
    vm.$watch('number', function(newValue, oldValue) {
        console.log(newValue, oldValue)
    })
</script>

# 计算属性

计算属性实际上也是函数，但是不需要加圆括号。值可以从函数中return，就自动进行计算。

// 模板内的表达式非常便利，但是设计它们的初衷是用于简单运算的。
// 在模板中放入太多的逻辑会让模板过重且难以维护
// 应当使用计算属性
<div id="example">
    {{ message.split('').reverse().join('') }}
</div>
// 使用计算属性
<div id="example">
    {{ reversedMessage }}
</div>
// computed是一个对象
data: {
    message: 'nihao'
},
computed: {
    reversedMessage() {
        return this.message.split('').reverse().join('')
    }
},

计算属性和方法的区别： 计算属性是基于依赖进行缓存的，只有依赖的值发生变化再回重新计算。

# v-if指令和v-show指令

React中的jsx中：

{a === true && <my-component>a为真显示</my-component>}

相应的vue中使用指令v-if来决定是否渲染组件，执行组件的生命周期等。

<span v-if='a'>a</span>
<span v-else='a'>!a</span>

可以用<template></template>标签来包裹一系列元素，上树的时候template元素不会上树，仅仅里面的元素上树：

<ul>
    <template v-for="item in items">
        <li>{{ item.msg }}</li>
        <li class="divider"></li>
    </template>
</ul>

而v-show指令则类似于display:none/block的作用，控制组件生成的DOM的展示或者隐藏。需要注意，template上面不能写v-show指令。

<span v-for='item in list' :key="item.id">

# 事件

绑定事件是直接绑定给真实DOM；

如果需要使用原生event对象，需要传入$event对象：

<div id="box">
<div class="statics" @mousemove='moveHandler($event)'></div>
<span>
    {{thisX}}, {{thisY}}
</span>
</div>
<script src='./node_modules/vue/dist/vue.min.js'></script>
<script>
    var vm = new Vue({
        el: '#box',
        data() {
            return {
                thisX: 0,
                thisY: 0,
            }
        },
        methods: {
            moveHandler(event) {
                this.thisX = event.clientX
                this.thisY = event.clientY
            }
        },
    })
</script>

事件相关：

<!-- 阻止默认事件 -->
<div @click.prevent='clickHandler'></div>
<!-- 阻止事件冒泡 -->
<div @click.stop='clickHandler'></div>
<!-- 点击键盘，回车 -->
<div @keyup.enter='clickHandler'></div>
<!-- 点击键盘，13编号键 -->
<div @keyup.13='clickHandler'></div>

# v-model

可以用v-model指令，在表单元素上进行进行数据双向绑定。v-model会忽略表单元素的value、checked、selected的初始值，使用Vue实例的数据。

默认值应该在组件的data选项中声明。

<input v-model="message" placeholder="edit me">
<!-- 多行文本 -->
<textarea v-model="message" placeholder="add multiple lines"></textarea>
<!-- 复选框 -->
<input type="checkbox" id="checkbox" v-model="checked">
<label for="checkbox">{{ checked }}</label>
<!-- 单选按钮，使用value来存值 -->
<div id="example-4">
    <input type="radio" id="one" value="One" v-model="picked">
    <label for="one">One</label>
    
    <input type="radio" id="two" value="Two" v-model="picked">
    <label for="two">Two</label>

    <span>Picked: {{ picked }}</span>
</div>
<!-- 下拉选框，multiple按住command可以多选，去掉该属性则时下拉单选 -->
<!-- 下拉菜单中，直接给select表单元素进行v-model的绑定，option不用绑定。 -->
<!-- 如果option身上没有value属性，不要紧，会自动使用option之间的innerHTML值。 -->
<!-- 如果有value，vue会使用这个value。 -->
<div id="example-6">
    <select v-model="selected" multiple style="width: 50px;">
    <option>A</option>
    <option>B</option>
    <option>C</option>
    </select>
    
    <span>Selected: {{ selected }}</span>
</div>

<script>
    var vm = new Vue({
        el: '#box',
        data() {
            return {
                message: '',
                checked: '',
                picked: '',
                selected: []
            }
        },
    })
</script>

# 组件(重点)

// 创建一个组件
var MyComponent = Vue.extend({
    template: "<div>自定义组件</div>"
})
// 注册全局组件
Vue.component('my-component', MyComponent)
new Vue({
    el: '#box'
})
// 注册全局组件
// 或者
Vue.component('my-component', {
    template: "<div>自定义组件</div>"
})
new Vue({
    el: '#box'
})

局部注册：

// 创建一个组件
var MyComponent = Vue.extend({
    template: "<div>自定义组件</div>"
})
// 局部组件
new Vue({
    el: '#box',
    components: {
        'my-component': MyComponent
    }
})

new Vue()和Vue.extend()的功能一样，返回一个组件类。new Vue()需要提供el属性，设置Vue运行的环境；Vue.extend()需要提供的template属性，内嵌组件。

组件类的data属性必须是一个函数，而不是一个JSON。因为每个实例需要维护一份被返回对象的独立的拷贝。

props属性表示期望从父组件传下来的数据，可以设置默认值。 如果prop以一种原始的值传入且需要进行转换，在这种情况下，最好使用这个prop的值来定义一个计算属性：

new Vue({
    el: '#box',
    data() {
        return {    
            a: 9999
        }
    }，
    // 子组件中用props接收
    props: {
        width: Number,
        height: Number,
        // 可以设置默认值
        tip: {
            type: String,
            default: 'hello'
        }
    }   
})
// 父组件中使用v-bind:或者:来传递数据
<parent-node :width='1000' v-bind:height= '800'></parent-node>

<!-- 动态组件 -->
<component v-bind:is="currentTabComponent"></component>
<script>
    export default{
        data() {
            return {
                currentTabComponent: ** // 已注册组件的名字/一个组件的选项对象
            }
        }
    }
</script>

# 组件间通信

Vue封装了父亲的实例、儿子的实例的方法，简化了通信这个事情。

  • 子组件可以用this.$parent访问它的父组件的js实例（是Vue对象，不是DOM元素）。
  • 根实例的后代可以用this.$root访问它（是Vue对象，不是DOM元素）**。
  • 父组件有一个数组 this.$children，包含它所有的子元素。

尽管可以访问父链上任意的实例，不过子组件应当避免直接依赖父组件的数据，尽量显式地使用props传递数据。

不要在子组件中修改父组件的状态：

  • 这让父组件与子组件紧密地耦合；
  • 只看父组件很难理解父组件的状态，因为它可能被任意子组件修改！理想情况下，只有组件自己能修改它的状态;

父组件通常的做法是通过this.$refs来访问子组件的方法和属性，也可以通过this.$children。反过来也有this.$parent提供给自组件来访问父组件的实例；

# .sync

开发中经常需要对一个prop进行"双向绑定"。

  • 组件内可以使用.sync;
  • 涉及父子组件，父组件可以使用.sync，子组件使用emit：
<!-- 子组件中 emit一个事件+值； -->
methods:{
    clickHandler() {
        this.$emit('update:title', newTitle)
    }
}
<!-- 父组件中，v-on监听那个事件并根据需要更新一个本地的数据属性 -->
<text-document
    v-bind:title="doc.title"
    v-on:update:title="doc.title = $event"
></text-document>
<!-- 简写 -->
<text-document
    :title="doc.title"
    @update:title="doc.title = $event"
></text-document>
<!--  等同于： -->
<text-document v-bind:title.sync="doc.title"></text-document>

# 自定义事件

自组件传递事件给父组件，或者说自组件出发写在父组件中的事件：

<!-- 子组件中 -->
methods:{
    clickHandler() {
        this.$emit('update', newTitle)
    }
}
<!-- 父组件中用v-on:来接收 -->
<text-document v-bind:update="doSomething"></text-document>
<!-- 简写 -->
<text-document @update="doSomething"></text-document>

# slot插槽

Vue 实现了一套内容分发的 API，这套 API 基于当前的 Web Components 规范草案，将 <\slot> 元素作为承载分发内容的出口。

<!-- 父组件中 -->
<navigation-link url="/profile">
Your Profile
</navigation-link>
<!-- navigation-link组件中
当组件渲染的时候，这个 <slot> 元素将会被替换为''Your Profile"
插槽内可以包含任何模板代码，包括 HTML：
甚至其它的组件
如果 <navigation-link> 没有包含一个 <slot> 元素，则任何传入它的内容都会被抛弃。 -->
<a v-bind:href="url" class="nav-link">
    <slot></slot>
</a>

有些时候我们需要多个插槽。例如，一个假设的 <base-layout> 组件多模板如下：

<div class="container">
  <header>
    <!-- 我们希望把页头放这里 -->
  </header>
  <main>
    <!-- 我们希望把主要内容放这里 -->
  </main>
  <footer>
    <!-- 我们希望把页脚放这里 -->
  </footer>
</div>

// 对于这样的情况，<slot> 元素有一个特殊的特性：name。这个特性可以用来定义额外的插槽：
<div class="container">
    <header>
        <slot name="header"></slot>
    </header>
    <main>
        <slot></slot>
    </main>
    <footer>
        <slot name="footer"></slot>
    </footer>
</div>
// 在向具名插槽提供内容的时候，我们可以在一个父组件的 <template> 元素上使用 slot 特性：
// 也可以直接用在一个普通的元素上：
// <slot> 标签内部可以指定默认的内容。
<base-layout>
    <template slot="header">
        <h1>Here might be a page title</h1>
    </template>
    <p>A paragraph for the main content.</p>
    <p>And another one.</p>
    <template slot="footer">
        <p>Here's some contact info</p>
    </template>
</base-layout>

# 自定义指令、自定义过滤器

自定义指令，除了内置指令，Vue.js 也允许注册自定义指令。自定义指令提供一种机制将数据的变化映射为 DOM 行为。可以用 Vue.directive(id, definition) 方法注册一个全局自定义指令，它接收两个参数指令 ID 与定义对象。也可以用组件的 directives 选项注册一个局部自定义指令。

// 注册指令
Vue.directive('alert', function(node) {
    alert(node.innerHTML)
})
new Vue({
    el: '#box',
    data() {
        return {
            number: '9999'
        }
    },
})

自定义过滤器：

<div id="box">
    <div v-alert>{{number | reverse}}</div>
</div>
<script src='./node_modules/vue/dist/vue.min.js'></script>
<script>
    // 注册指令
    Vue.directive('alert', function(node) {
        alert(node.innerHTML)
    })
    // 注册指令
    Vue.filter('reverse', function(value) {
        return value.split('').reverse().join('')
    })
    new Vue({
        el: '#box',
        data() {
            return {
                number: 'nihao'
            }
        },
    })
</script>
