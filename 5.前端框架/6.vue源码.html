# 6.vue源码

# 主要分为两个系统

  1. 数据驱动，即数据系统，数据响应式。主要包括数据的劫持、发布订阅等，具体体现在Observer(数据劫持)、Dep(收集依赖，通知Watcher)、Watcher(订阅者，包括render-watcher、computed-watcher、normal-watcher)三个类；

  2. 组件化，即编译系统，虚拟DOM。主要包括模版解析、静态节点标记优化、生成可渲染的字符串，具体体现在paser(生成AST)、optimize(标记静态节点)、generate(生成渲染函数)三个方法。

# 核心实现-响应式

  1. 创建Vue类，通过Observer劫持监听所有属性。

observe函数的作用：递归遍历data选项，它当中的defineReactive函数为data中每一个key定义getter和setter，达到数据劫持的目的。

    1. 创建Dep类，将视图中的所有依赖收集管理，包括依赖添加和派发通知；
deps=[]; // 存放Watcher的实例
addDep; // 添加Watcher
notify; // 通知所有的Wather执行更新。遍历deps数组，调用每个Wather的更新方法
    1. 创建监听器Watcher类;

  • 收集：在created和beforeMounted之间生成了AST和render可执行函数，用到了compile的createCompiler生成对象结构的AST语法树，再通过generator调用了之前初始化的方法处理各种vue内置的方法&指令，如v-for v-if等，通过createElement、createTextVnode等生成虚拟DOM树，作为updateComponent函数返回。

  • 渲染：在beforeMounted和mounted之间通过将执行这个函数，这个函数会将虚拟DOM渲染成真正的DOM，内含有patch的布丁算法，在执行过程中会触发渲染函数，这时会触发数据属性的get函数，而这个过程的观察者就是渲染函数，在渲染过程中添加数据的订阅者，watcher订阅者是Observer和Compiler之间的桥梁，将在自身实例化的时候将自己添加到dep当中，watcher自身有一个depend和update方法，待属性变动dep.notice通知调用自身的update方法触发Compile中绑定的回调。

Dep.target = this; // 将当前Watcher实例添加到Dep.target上。get时，就能通过Dep.target拿到当前Watcher的实例。
update;
set; // 调用dep.notify，让依赖管理器通知更新，则所有的Watcher会执行update方法

  1. 创建compile.js，用于扫描模板中所有依赖(指令、插值、绑定、事件…)，创建更新函数和Watcher；

获取所有的孩子节点，进行遍历，判断节点类型，并作出相应的判断; 处理元素节点，遍历可能存在的子节点，往下递归;

// 模版编译

# 模版编译

主要是在compile过程中：

  • parse函数解析template;
  • optimize函数优化静态内容；
  • generate函数创建render函数字符串，等待执行vdom时调用，进入patch流程，最终上DOM树；
  1. parse函数解析最终生成AST树：

AST，在VUE中指的是一个嵌套的、有标签名、属性、父子关系等的JS对象，以树形来表示DOM结构。

  1. optimize优化：

parse出来的AST进行标记静态节点、非静态节点的标记

静态节点，指的是没有订阅state，不需要每次都刷新的标签

  1. generate生成render函数：

输入AST，返回AST，对AST进行递归，调用相应节点生成标签字符串的方法，返回生成字符串的render函数

# Patch

对比新旧的Vnode，创建、删除或者更新DOM节点/组件实例；使用Diff算法尽可能的复用先前的DOM节点；

# vue3

Vuejs3 在北京时间2020年9月19日凌晨，终于发布了3.0版本，代号：One Piece；

重要变更：

  • 数据响应式使用proxy(1);
  • Tree-shaking support （尽量使用更有利于Tree-shaking的 export const/function 写法。4）；
  • Composition API (setup入口，方便业务解耦合。2)；
  • Fragment（支持多个根结点。5）, Teleport（可以任意地丢到html中的任一个DOM下）, Suspense（控制异步组件）等新的玩法；
  • Better TypeScript support （更好的 TypeScript 支持度。3）；
  • Custom Renderer API （自定义的 Renderer API createRenderer）；
  • 解耦内部模块，例如可以单独使用reactivity（6）

性能上，有以下改进：

  • Rewritten virtual dom implementation （重写了虚拟 DOM）；
  • Compiler-informed fast paths （优化编译）；
  • virtual dom静态节点的hosting，真正的virtual node只包含动态节点；
  • More efficient component initialization （更高效的组件初始化）；
  • 1.3-2x better update performance （1.3~2 倍的更新性能）；
  • 2-3x faster SSR （2~3 倍的 SSR 速度）；
  • vite (构建工具)

Oject.defineProperty的缺陷：

  1. 无法监测到对象属性的添加和删除；
  2. 无法直接监听数组基于下标的变化，
  3. vue2重写了push pop shift unshift splice sort reverse等会直接修改原数组的操作
