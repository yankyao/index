# 核心实现-响应式

  1. 创建Vue类，通过Observer劫持监听所有属性。

observe函数的作用：递归遍历data选项，它当中的defineReactive函数为data中每一个key定义getter和setter，达到数据劫持的目的。

    1. 创建Dep类，将视图中的所有依赖收集管理，包括依赖添加和派发通知；
deps=[]; // 存放Watcher的实例
addDep; // 添加Watcher
notify; // 通知所有的Wather执行更新。遍历deps数组，调用每个Wather的更新方法
    1. 创建监听器Watcher类;
Dep.target = this; // 将当前Watcher实例添加到Dep.target上。get时，就能通过Dep.target拿到当前Watcher的实例。
update;
set; // 调用dep.notify，让依赖管理器通知更新，则所有的Watcher会执行update方法

  1. 创建compile.js，用于扫描模板中所有依赖(指令、插值、绑定、事件…)，创建更新函数和Watcher；

    1. 创建编译器Compile类，接收两个参数，el(宿主元素或选择器)和vm(当前vue实例);
    1. 创建node2Fragment函数，将dom节点( $el )截成代码块( 转换为Fragment )来处理，而不是直接做dom操作，提高执行效率。

创建一个新的fragment，将原生节点移动至fragment; 返回fragment，传给编译函数进行编译；

    1. 创建compile函数，执行编译( 将模板中的动态值替换为真实的值 )，传入代码块;

获取所有的孩子节点，进行遍历，判断节点类型，并作出相应的判断; 处理元素节点，遍历可能存在的子节点，往下递归;

// 模版编译
compileElement; // 编译元素节点，拿到所有属性名称，进行遍历
compileText; // 编译文本节点
    1. 将生成的结果插入el;

# 模版编译

主要是在compile过程中：

  • parse函数解析template;
  • optimize函数优化静态内容；
  • generate函数创建render函数字符串，等待执行vdom时调用，进入patch流程，最终上DOM树；
  1. parse函数解析最终生成AST树：

AST，在VUE中指的是一个嵌套的、有标签名、属性、父子关系等的JS对象，以树形来表示DOM结构。

  1. optimize优化：

parse出来的AST进行标记静态节点、非静态节点的标记

静态节点，指的是没有订阅state，不需要每次都刷新的标签

  1. generate生成render函数：

输入AST，返回AST，对AST进行递归，调用相应节点生成标签字符串的方法，返回生成字符串的render函数

# Patch

对比新旧的Vnode，创建、删除或者更新DOM节点/组件实例；使用Diff算法尽可能的复用先前的DOM节点；

# vue3

Vuejs3 在北京时间2020年9月19日凌晨，终于发布了3.0版本，代号：One Piece；

有 6 个方面的重要变更：

  • Performance （性能）；
  • Tree-shaking support （支持 Tree-shaking）；
  • Composition API；
  • Fragment, Teleport, Suspense；
  • Better TypeScript support （更好的 TypeScript 支持度）；
  • Custom Renderer API （自定义的 Renderer API）；

性能上，有以下改进：

  • Rewritten virtual dom implementation （重写了虚拟 DOM）；
  • Compiler-informed fast paths （优化编译）；
  • More efficient component initialization （更高效的组件初始化）；
  • 1.3-2x better update performance （1.3~2 倍的更新性能）；
  • 2-3x faster SSR （2~3 倍的 SSR 速度）；
