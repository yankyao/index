  10. class：新的类的书写方式，静态方法、super、extends；

  14. 模块：import/module；
  15. Generator 函数;

  1. async、await：异步解决新方案；

# es2020

  1. 可选链、双问号
const a = {
    base: {
        name: 3,
        tag: ['boy'],
        index: 0,
    }
}
console.log(a?.base?.name) // 3
console.log(a?.base?.age || 10) // 10
console.log(a?.base?.index || 10) // 10
console.log(a?.base?.index ?? 10) // 0

# 模块机制

  1. CMD和ESModule的区别：
1. ESModule的采用静态编译，使得编译时就能确定模块的依赖关系，以及输入和输出的变量。前者在导出时都是值拷贝，只能在运行时确定。就算导出的值变了，导入的值也不会改变，所以如果想更新值，必须重新导入一次。但是后者采用实时绑定的方式，导入导出的值都指向同一个内存地址，所以导入值会跟随导出值变化；
2. 前者支持动态导入 require(${path}/xx.js)，后者目前不支持。
3. 前者是同步导入，因为用于服务端，文件都在本地，同步导入即使卡住主线程影响也不大。而后者是异步导入，因为用于浏览器，需要下载文件，如果也采用同步导入会对渲染有很大影响。
