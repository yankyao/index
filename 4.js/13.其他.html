# 事件环Event Loop

JS运行机制：JS执行的过程基本是单线程的，所有的任务都是在一条线程上； JS通过Event Loop模型来处理异步和回调；

  • 堆（Heap），堆是一种数据结构，是利用完全二叉树维护的一组数据，堆分为两种，一种为最大堆，一种为最小堆，将根节点最大的堆叫做最大堆或大根堆，根节点最小的堆叫做最小堆或小根堆。堆是线性数据结构，相当于一维数组，有唯一后继。

  • 栈（Stack），栈在计算机科学中是限定仅在表尾进行插入或删除操作的线性表。栈是一种数据结构，它按照后进先出的原则存储数据，先进入的数据被压入栈底，最后的数据在栈顶，需要读数据的时候从栈顶开始弹出数据。

  • 队列（Queue），队列的数据元素又称为队列元素。在队列中插入一个队列元素称为入队，从队列中删除一个队列元素称为出队。因为队列只允许在一端插入，在另一端删除，所以只有最早进入队列的元素才能最先从队列中删除，故队列又称为先进先出。

Event Loop分为：

  • 宏任务（MacroTask），setTimeout、setInterval、setImmediate（浏览器暂时不支持）、I/O、UI Rendering。
  • 微任务（MicroTask），Process.nextTick（Node独有）、Promise、Object.observe(废弃)、MutationObserver。

# 事件循环的进程模型

  • 选择当前要执行的任务队列，选择任务队列中最先进入的任务，如果任务队列为空即null，则执行跳转到微任务（MicroTask）的执行步骤。
  • 将事件循环中的任务设置为已选择任务，执行任务。
  • 将事件循环中当前运行任务设置为null，将已经运行完成的任务从任务队列中删除。
  • microtasks步骤：进入microtask检查点。

# 滚屏

以''页面"为单位进行切换，页面上的元素有进场、出场动画。页面没有纵向滚动条，纵向滚动页面失效。

可以监听mousewheel事件，用container的top值来驱动整个屏幕运动。

<html lang="en">
<head>
    <title>Document</title>
    <style type='text/css'>
        * {
            margin: 0;
            padding: 0;
        }
        html, body, .container{
            height: 100%
        }
        body {
            overflow: hidden;
        }
        .container {
            position: relative;
            top: -100%;
        }
        .page{
            width: 100%;
            height: 100%;
        }
        .page2 {
            background-color: aquamarine
        }
    </style>
</head>
<body>
    <div class="container">
        <div class="page1 page"></div>
        <div class="page2 page"></div>
        <div class="page3 page"></div>
        <div class="page4 page"></div>
        <div class="page5 page"></div>
    </div>
    <script>
        // 鼠标滚轮事件
        var idx = -1
        $(document).mousewheel(function(event, direction) {
            // 设置老的信号量
            var oldIdx = idx
            idx = -direction
            if(idx < -1) {
                idx = -1
            } else if(idx > 4) {
                idx = 4
            }
            // 用数组来存储动画函数,执行页面的进出场动画
            enter[idx]()
            exit[oldIdx]()
            // 头部header和nav的判断
            if(oldIdx === -1 && idx === 0) {
                $('.header').slideUp(function() {
                    $('.nav').slideDown()
                })
            }
            if(oldIdx === 0 && idx === -1) {
                $('.nav').slideUp(function() {
                    $('.header').slideDown()
                })
            }
            if(idx === -1 || idx === 4) return
            $('.container').stop(true).animate({'top': -idx * 100 + '%'}, 600)
        })
    </script>
</body>
</html>

# 抛出错误

throw内置对象Error，会终止JS的运行;

function errorWrap(bool) {
    try{
        if(bool) {
            console.log('true')
        } else {
            throw new Error('false')
        }
    } catch(err) {
        console.log(err)
    } finally {
        console.log('执行完毕')
    }
}
errorWrap(true)
errorWrap(false)

# Cookie和Session

Cookie，是Response头部的一个键值对（K:V）。

  • 当服务器下发了一个Set-Cookie（比如需要输入密码之类的），后面再一定的时间内浏览器浏览相同的网页，发送的Request都会带着相同的Cookie上去；HTTP是无连接的，当你第一次访问一个服务器的时候，第二次再来到这个服务器（可能就10s之后），服务器此时的HTTP和第一次毫无关系（通过Cookie连接服务器的间接关系）的。对于服务器来说，HTTP是两次，两次之间没有持久保持连续。(http1.1是这样，http2.0有优化)

  • 服务器setcookie函数就是设置Cookie，有三个参数：K、V、过期时间.

/* 用js封装一个cookie操作 */
function setCookie(key, value, t){
	var oDate=new Date();
    oDate.setDate(oDate.getDate()+t);
    /* 可以设置过期时间 expires= */
    /* 存一个编码后的字符串 */
    /* document.cookie=表示添加或者修改，并不会清除非key的值 */
	document.cookie=key+"="+encodeURIComponent(value)+";expires="+oDate.toUTCString();
	if(getCookie(key)){
		alert('设置成功');
	}else{
		alert('设置失败');
	}
}
function getCookie(key){
	var str=document.cookie.replace(/;\s*/,';');
	var cookieArr=str.split(';');
	var cookieObj={};
	var len=cookieArr.length;
	for(var i=0; i<len; i++){
		var item=cookieArr[i];
		var k=item.split('=')[0];
		var v=item.split('=')[1];
		cookieObj[k]=v;
	}
	if(cookieObj[key]){
        /* 获得cookie的时候，得到的是一个字符串。如果是编码后的，要进行解码 */
		return decodeURIComponent(cookieObj[key]);
	}else{
		return false;
	}
}
function removeCookie(key){
	setCookie(key, ' ', -1);
	if(!getCookie(key)){
		alert('删除成功');
	}else{
		alert('删除失败');
	}
}

Session，服务器端记录登陆的标识。Session的实质是服务器一种记录客户状态的机制，服务器不发送明文，而是向客户端发送随机乱码的cookie。（这随机乱码就是身份证，有时效性）在服务器端建立客户状态记录。
