# 3.函数

# 函数的声明和调用

function是引用类型值；

函数和变量一样，必须先声明（定义）才能使用。

函数名是一种JS标识符，需要遵循相关的命名规则，首字母必须是字母、下划线、$，后面的字符也可以是数字；区分大小写，并且不能是关键字和保留字。

函数的声明，定义了函数执行时要执行的代码块，需要调用才会执行内部的代码（在没有调用执行的情况下，结构体内的代码维持不变；在执行的情况下，才会依次解析并执行结构体内的代码）。

var a = [];
for(var i = 0; i < 10; i++) {
    a[i] = function() {
        console.log(i)
    }
}
a[8]()  // 10,使用var定义的是全局变量，每调用一次，i就会重新赋值。函数没执行，而是存在数组中，调用的时候总是调用最后的i的值10。
var a = [];
for(let i = 0; i < 10; i++) {
    a[i] = function() {
        console.log(i)
    }
}
a[8]() // 8，使用let定义的变量，仅在块级作用域中有效，函数的闭包特性，记住了每次生成的作用域，使得每次let定义的值都存在内存中。

函数调用，直接在函数名后面加小括号。函数内部的结构体执行的位置，跟定义的位置无关，跟函数调用的位置有关；函数声明一次，可以进行多次调用，每次调用内部的代码结构体都会从新执行。

# 函数的参数

  1. 定义函数时，在小括号内部定义的变量，成为形式参数；
  2. 调用时，小括号内是实际参数，给形式参数传输实际参数，实际参数参与内部的代码执行的过程。
  • 形式参数可以理解为变量，可以有一个或者多个形式参数；实际参数可以是数据字面量，也可以是变量。
  • 函数的实际参数和形式参数的个数可以不同，实参多于形参，多出来的实参，不参与程序的执行；实参少于形参，多出的形参没有赋值，默认的是undifined值，参与程序的执行。
  • 函数内部的arguments是实际参数组成的一个类数组对象。
    • arguments.callee.length 形参列表的个数。
    • arguments.length 实参个数，就是调用函数的时候传进来的实参个数。

# 函数的特点

  • 函数能够封装一段代码，可以重复的使用，使用时直接调用函数即可；

  • 函数可以有返回值，可以将他的返回值当作另一个函数的变量，继续参与编程；

函数可以通过执行内部代码改变数据，也可以通过return返回一个值，返回的值可以是基本类型数据，也可以是另一个函数，也可以是其本身。

函数内部运行到return，就不会再继续执行，而是跳出函数。

# 模块化编程

将编程的一些通用的方法、过程封装(设定好通用参数，回调函数，return业务函数)到函数里，从返回值返回结果/业务函数，总体的业务编程可以重复使用前面封装的函数。

多个程序可以调用同一段代码，公用的部分就可以封装为函数。

# 函数递归

函数内部调用自己本身，例如斐波那契函数；实现递归的方式有几种：

  • 通过函数名或者函数表达式；
  • 通过arguments.callee，arguments.callee就是函数自己；
  • 通过对象方法等。

# 函数表达式

通过将一个函数（可以匿名，也可以有函数名）矮化成一个表达式，赋值给一个变量的形式定义函数；可以使用变量名加小括号进行调用。

有函数名的函数不能使用自身的函数名进行调用，会报错；但可以在函数自己的内部调用它的函数名。例如：

var fun = function fun2(a) { console.log(a) }
fun(2)  // 2
fun2(2) // Uncaught ReferenceError: fun2 is not defined

WARNING

函数表达式不存在函数声明提升，只有变量声明提升；

函数可以先使用后定义，js解析器在运行的时候，会将所有的全局变量提升到最开始的位置。使用的时候，可以根据变量名找到这个函数的整体原型（区别，变量的声明提升，只提升声明，不提升赋值）。函数表达式不会进行函数的声明提升，因为函数表达式相当于，将函数赋值给一个变量。

# 函数作用域

在函数内部定义或者声明的变量，不能在函数之外的地方获取，这样的变量在函数内部才有定义，因为函数限定了变量的作用域。

  • 在JS中，函数是实现变量作用域的唯一结构；
  • 在函数内部定义的变量，是这个函数的局部变量，只能在该函数内部调用，在函数外部调用会报错；
  • 在函数外部定义的变量，是这个函数或者与函数平级的函数的全局变量，是一个相对概念；
  • 在作用域函数的嵌套中，函数内部再调用一个变量时，会从自己的作用域依次往外层查找，用的是第一个匹配到的声明的值。

TIP

  • 定义变量时，没用使用var关键字，JS默认是全局变量，会造成全局污染;
  • 变量一般都用var关键字来定义，传递（信号量、中间量）、累积（累加、累乘）一般用var定义在当前函数的外层。

# 函数重载

函数重载：在其它语言中，一个函数定义两次，每次有不同数量的形式参数，代表的是两个不同的函数，叫做函数的重载overloaded。

JS语言中没有函数重载的现象：一个函数名只能指向一个函数，在同一个作用域内，后声明的函数会覆盖前面声明的函数。

JS函数中有一个arguments的类数组对象。在函数的调用过程中，传递的参数无论有多少个，也不论实际参数有没有对应的形式参数，都会以单独数据的形式存到arguments对象内部。

TIP

  • arguments做为类数组对象，不能使用数组的所有方法，很多可以使用的例如：length，下标遍历等等。
  • arguments可以用于封装函数是模拟函数重载现象，或者检测用户输入函数的数据的个数或者形式等等。

# IIFE

即时调用函数表达式，immediately-invoked function expression。指的是在函数的声明、定义的同时，直接调用。两种实现方法：将函数矮化成表达式，加小括号调用；常用的是小括号包裹函数，后面再加小括号直接调用。

IIFE的函数不能再在外面使用函数名进行调用，因为IIFE将函数声明在自己作用域内，在外面找不到这个函数；最后面的直接调用小括号里可以传参数。

作用：

  • 解决全局污染问题，可以暴露一个接口，这样一个IIFE就成了一个独立的模块；
  • 异步语句与闭包。
(function(a){
    console.log(a)
})(2)         // a

var array = []
for(var i = 0; i < 10; i++) {
    (function(i) {
        array[i] = function() {
            console.log(i)
        }
    })(i)
}
array[3]          //3

for(var i = 0; i < 10; i++) {
    (function(i) {
        setTimeout(function(){
            console.log(i)        // 输出0-9
        }, 100)
    })(i)
}

函数定义一次可以多次调用，每次调用都会产生新的闭包。

# 函数的上下文

即this的指向（用function关键字声明的函数）：

  • 函数加括号直接运行，不带前缀，函数的上下文为window对象；
  • 函数做为对象的方法，对象打点加括号调用，函数的上下文指的是这个对象；
  • 函数是事件处理函数，函数的上下文指的是触发这个事件的DOM元素；
  • 定时器调用函数(回调函数)，函数本身的上下文为window对象（定时器调用函数里面的函数，要再次区分这五种情况；）；
  • 数组中存放函数，被数组索引之后加括号调用，函数的上下文为这个数组；
  • new的this指的是四步走的空对象；

call、apply、bind，是Function对象自带的三个方法。

  • apply和call都是为了改变某个函数运行时的上下文，this指向他们的第一个参数；
/* 二者区别： */
fun.apply(context, array);
fun.call(context, param1, params2);

const funBind = fun.bind(context);
funBind()

bind与apply/call的区别：

  • bind不会立即调用，其他两个会立即调用。
  • bind会创建一个新函数，称为绑定函数，当调用这个函数的时候，绑定函数会以创建它时传入bind()方法的第一个参数作为this，传入bind()方法的第二个及以后的参数加上绑定函数运行时本身的参数按照顺序作为原函数的参数来调用原函数。

# 闭包closure

闭包不是特殊的结构，一个函数就是一个闭包。函数在定义时，会形成自己的作用域。外部环境和内部语句组成了一个完整的环境，就是闭包。

/* 非闭包 */
function init() {
    var name = "Mozilla"; // name 是一个被 init 创建的局部变量
    function displayName() { // displayName() 是内部函数，一个闭包
        alert(name); // 使用了父函数中声明的变量
    }
    displayName();
}
init();
/* 闭包 */
function makeFunc() {
    var name = "Mozilla";
    function displayName() {
        alert(name);
    }
    return displayName;
}

var myFunc = makeFunc();
myFunc();

# 高级函数/高阶函数

高阶函数英文叫Higher-order function。接收一个函数作为参数，并且常常返回一个函数，进行参数包装、属性继承。例如react的高阶组件。

function add(x, y, f) {
    return f(x) + f(y);
}

典型的函数柯里化，函数柯里化指的是将能够接收多个参数的函数转化为接收单一参数的函数，并且返回接收余下参数且返回结果的新函数的技术。主要作用和特点就是参数复用、提前返回和延迟执行：

/* 典型的bind就是一个柯里化的函数 */
fun.bind(null, args);
/* 最简单的例子 */
function currying(fn, ...args) {
    return function(...args2) {
        fn.apply(null, args.concat(args2))
    }
}

function add(x, y) {
    return x + y
}
// Currying后
function curryingAdd(x) {
    return function (y) {
        return x + y
    }
}

add(1, 2)           // 3
curryingAdd(1)(2)   // 3

function carry(fn, ...args) {
     // 获取外层第一个参数后的所有参数
     const _this = this;
     const totalArgs = args || [];
     const len = fn.length; // 形参的个数

     return function () {
        const innerargs = Array.prototype.slice.call(arguments);
        args.push(...innerargs);

        if(args.length < len) {
            return carry.call(_this, fn, args);
        }
        return fn.apply(null, args)
     }
}
// 实现add(1, 2, 3, 4)=10,add(1)(2)(3)(4)(5)=15,add(1, 2, 3, 4)(5)(6)=21
function add(...args) {
    function inner(...innerArgs) {
        args = [...args, ...innerArgs];
        return inner;
    }
    inner.toString = function() {
        return args.reduce(function (res, i) {
            return res + i
        })
    }
    return inner
}

提前返回，避免每次都进行逻辑判断。

/* 提前返回 */
var on = function(element, event, handler) {
    if (document.addEventListener) {
        if (element && event && handler) {
            element.addEventListener(event, handler, false);
        }
    } else {
        if (element && event && handler) {
            element.attachEvent('on' + event, handler);
        }
    }
}
/* 不用每次都进行document.addEventListener是否支持的判断 */
var on = (function() {
    if (document.addEventListener) {
        return function(element, event, handler) {
            if (element && event && handler) {
                element.addEventListener(event, handler, false);
            }
        };
    } else {
        return function(element, event, handler) {
            if (element && event && handler) {
                element.attachEvent('on' + event, handler);
            }
        };
    }
})();

# Thunk函数

在 JavaScript 语言中，Thunk 函数替换的不是表达式，而是多参数函数，将其替换成一个只接受回调函数作为参数的单参数函数。

const Thunk = function(fn){
    return function(...args) {
        return function(callback) {
            return fn.call(this, ...args, callback);
        }
    }
}
/* fs模块的readFile方法是一个多参数函数，两个参数分别为文件名和回调函数。 */
const readFileThunk = Thunk(fs.readFile);
readFileThunk(fileUrl)(callback);
