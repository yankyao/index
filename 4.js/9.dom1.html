# 9.DOM1

# 概述

Document Object Model，文档对象模型。DOM描绘了一个层次化的节点树，允许开发人员添加、修改、移除页面的某一部分；使得JS操作HTML时，不是在操作字符串，而是HTML的节点。 历史：98年之前的成为DOM 0级规范，98年指定1级规范，现在更新到3级规范。 DOM通过节点的描述方式，提供了丰富的API：获取元素（节点）、css样式、事件、运动、元素的尺寸位移、节点操作等等。

# 获取元素

JS通过document对象获取元素，document表示文档，及整个html结构。学习DOM，就是学习document对象；DOM操作，是从获取HTML元素或者节点开始的，获取元素的方法有：

// 通过顶层document节点获取：
// 可以准确获得需要的元素，是比较简单快捷的方法。如果页面上含有多个相同id的节点，那么只返回第一个节点。（id唯一）
document.getElementById(elementId)
// 返回节点数组
document.getElementsByClassName(elementName)
// 返回节点数组
document.getElementsByTagName(tagName)

// 通过父节点获取：
// 已知节点（parentObj）的第一个子节点，可以递归使用
parentObj.firstChild
// 已知节点（parentObj）的最后一个子节点，可以递归使用
parentObj.lastChild
// 已知节点的子节点数组(所有node类型的)
parentObj.childNodes
// 已知节点的直接子节点数组(标签元素)
parentObj.children
// 已知节点的所有子节点中类型为指定值的子节点数组
parentObj.getElementsByTagName(tagName)

// 通过临近节点获取：
// 已知节点（neighbourNode）的前一个节点，可以递归使用
neighbourNode.previousSibling
// 已知节点（neighbourNode）的下一个节点，可以递归使用
neighbourNode.nextSibling

// 通过子节点获取：
// 获取已知节点的父节点
childNode.parentNode

操作HTML，在获取元素之后，得到的是对象型的数据，可以得到获得元素一些属性的属性值，也可以修改。有两种方法，一种是getAttribute( )和setAttribute("属性名","新属性" )，一种是直接使用点语法。

console.log(oImg.src)    // 在控制台输出元素的地址属性的属性值
console.log(oImg.id)     // 在控制台输出元素的id属性的属性值
console.log(oImg.alt)    // 在控制台输出图片的alt属性的属性值
// 操作DOM元素里的内容
//读取:
box.innerHTML
//更改：
box.innerHTML(content)

二者的区别：

  • HTML可以使用自定义属性，点语法不能调用和更改HTML自定义属性；
  • 点语法在调用HTML属性的时候，要规避一些关键字：class要改为className，for要改为htmlFor，rowspan要改为rowSpan，colspan要改为colSpan；get语法则不需要。
  • 点语法调用style属性，得到的是css对象（行内样式），可以继续调用css的样式，遇到复合属性的单一写法，要用驼峰命名法；get不需要，直接在后面的括号里写css内的命名即可。
  • 点语法调用的style是一个对象，get方法得到的style则是一个字符串，不能再打点调用。
  • 添加改变类名：打点调用className。
  • 一般工作中，除了自定义属性用get、set方法(input行内样式很多要用这种方法 )，其他的都用点语法。

操作CSS，点语法，调用style属性，得到的是行内样式，更改的也是行内样式。通过点语法更改css样式，是通过等号进行赋新值的过程，后面的新值要用双引号包裹。

元素.style.属性
box.style.background = 'red'

# 事件

事件监听：给元素添加事件，触发事件就会执行相应的程序。JS在加载的时候，会先看整个页面哪些元素绑定了事件，浏览器引擎就会去对这些元素进行监听，触发事件则执行相应的程序。

// DOM 0级事件：
onclick         // 单击事件
ondblclick      // 双击事件
onmouseover     // 鼠标移上事件
onmouseout      // 鼠标移出事件
onmousedown     // 鼠标按下事件
onmouseup       // 鼠标弹起事件
onfocus         // 获得焦点事件
onblur          // 失去焦点事件
onload          // 加载事件
oninput         // 输入事件（类似实时）

# 计算后样式

指的是最终显示在浏览器的样式，是元素的只读属性；在对元素进行一些判断的时候，一般用计算后样式，而不是用 box.style.background这种方式；计算后样式得到的都是字符串！

// 高级浏览器，包括IE9及以后的浏览器兼容：
window.getComputedStyle(box).getPropertyValue("width")  //（小括号内不能写驼峰语法）
window.getComputedStyle(box)["backgroundColor"]         //（只能写驼峰语法）

// 低版本浏览器，包括IE6/7/8：
元素.currentstyle.属性
元素.currentstyle["backgroundColor"]     // （只能写驼峰语法）

//封装一个函数，兼容高级浏览器和低版本浏览器
function fetchComputedStyle(ele,property){
    //判断是高级浏览器还是低版本
    //能力测试判断
    if(window.getComputedStyle){
        //针对用户输入的属性名进行兼容写法，不管是不是驼峰，都改成横线写法
        property = property.replace(/([A-Z])/g,function(match,$1){
            return "-" + $1.toLowerCase();
        });
        return window.getComputedStyle(ele)[property];
    }else if(ele.currentStyle){
    //针对用户输入的属性名进行兼容写法，不管是不是驼峰，都改成驼峰
        property = property.replace(/\-([a-z])/gi,function(match,$1){
            return $1.toUpperCase();
        });
        return ele.currentStyle[property];
    }
}

# 快捷尺寸和位置

快捷尺寸和位置得到的是数字类型的数据，直接打点调用。 offsetLeft和offsetTop，offsetLeft参考的是元素天生的offsetParent，叫做偏移参考元素。有严重的兼容性问题：

// IE9及高级浏览器：
offsetParent：不论元素自身有没有定位，offsetParent为距离元素最近的有定位的祖先元素；祖先元素都没有定位，参考的是body；
offsetLeft：元素自身的左边框外到offsetparent的左边框内的距离；
offsetTop：元素自身的上边框外到offsetParent的上边框内的距离；
// IE6、7：
    ①元素自己没有定位：
    offsetParent：距离最近的有宽高的祖先元素，如果都没有宽高，参考body；
    offsetLeft：与高级浏览器一样，元素左边框外到offsetParent的左边框内部的距离；
    ②元素自己有定位：
    offsetParent：与高级浏览器一样，距离最近的有定位的祖先元素，没有的话参考body。
    offsetLeft：与高级浏览器一样，自身的左边框外到offsetParent的左边框内部的距离。
// IE8：
offsetParent：与高级浏览器选择方法一样。
offsetLeft：自身的左边框外到offsetParent的左边框外，多了一条offsetParent的左边框宽度。

offsetWidth和offsetHeight，与offsetParent没有关系。自身的实际占有的宽度和高度，全线浏览器兼容。特殊情况：

  • 不写宽度：会去继承父亲的宽度，得到的offsetWidth是具体的像素值。不会返回100%。
  • 不写高度：靠内部的内容撑高，具体的像素值。
  • 谷歌浏览器的加载机制：在加载图片的时候，不会等待，先去加载JS：得到撑高的高度时，没有图片撑高的部分，js加载完了之后，图片才会显示。

解决方法：window.onload事件去解决。

offsetWidth = width + padding + border
offsetHeight = height + padding + border

clientWidth和clientHeight，基本全线兼容，只有IE6有一点小问题，最常用的API。特殊情况：

  • 不写宽度：会去继承父亲的宽度，得到的clientWidth是具体的像素值。不会返回100%。IE６返回０.
  • 不写高度：靠内部的内容撑高，具体的像素值。IE6返回０.
clientWidth = width + padding
clientHeight = height +padding

# 原生JS方法操作DOM节点

// 创建节点，document.createElement(''标签名")，创建的是孤儿节点（孤儿节点也可以进行操作）；
var img = document.createElement('img')
// 添加在最后面
document.body.appendChild(img)
// 添加在最前面
document.body.prepend(img)
// 插入existNode之前
document.body.insertBefore(img, existNode)

如果括号里写一个已经在DOM树上的节点，其结果是移动这个节点到父亲里。新建节点里的内容可以用innerHTML来添加；特别的，创建IMG标签，可以用new Image（）；JS中存储的DOM节点的变量是动态的，当节点里面的元素发生变化时，变量里面的存的也会发生相应的变化。 移出节点，节点只能用他的父亲来删除它：

父亲.removeChild( 要移除的节点);
节点.parentNode.removeChild(节点); // 最常用的方法，removeChild方法有返回值，其返回值是移出的这个节点

替换节点：

父亲.replaceChild(新儿子,老儿子);

克隆节点，最简单的是innerHTML方法，但是这种方法是操作字符串来创建节点，效率很低；DOM创建节点的方法永远比字符串方法高；克隆出来的节点也是孤儿节点，需要上树。

节点.cloneNode() // 克隆这个节点，括号里可以传布尔值，为true则克隆这个节点的所有后代元素；

# 事件监听（注意兼容）

事件流，我们点击最内层的div，实际上系统会认为我们点击了所有的盒子，甚至于点击了body、点击了document、点击了 window。人们为了描述事件的传播，人为规定了一个事件的传播方向，称为事件流。两个阶段：事件捕获阶段、事件冒泡阶段。事件先从最大的元素往内层传播（捕获阶段），然后从最内层的元素往外层传播（冒泡阶段）。

// 点击box1会执行所有的
box1.onclick = function(){alert("我是box1");};
box2.onclick = function(){alert("我是box2");};
box3.onclick = function(){alert("我是box3");};
document.body.onclick = function(){alert("我是body");}
document.onclick = function(){alert("我是document");}
window.onclick = function(){alert("我是window");}
  • DOM0级事件监听，用on***的语法添加监听，称为DOM0级事件监听；
  • DOM0级事件监听只能监听冒泡阶段；DOM0级事件处理函数中，this指的是触发事件的DOM元素，就是事件传播到的这个元素；
  • DOM0级事件处理函数中，如果给同一个对象，同一个事件名，绑定多个监听，后写的覆盖先写的；
  • DOM0级事件监听，IE6、7、8中事件只能冒泡到document对象，不能继续冒泡到window对象了。也就是说，我们不能给window对象添加click事件监听。
  • DOM2级事件监听，DOM1级没有丰富事件监听，所以没有DOM1级的事情；DOM2中，W3C推出了addEventListener()函数。add''添加"，event''事件"，listener''监听"。
box1.addEventListener("click", function(){
    alert("我是box1的捕获阶段！");
}, true);
// true表示捕获阶段；false表示冒泡阶段。 口诀：true下false上；

// 看看执行顺序：
box1.addEventListener("click", function(){
    alert("我是box1的捕获阶段！");
}, true);
box2.addEventListener("click", function(){
    alert("我是box2的捕获阶段");
}, true);
box3.addEventListener("click", function(){
    alert("我是box3的捕获阶段");
}, true);
box1.addEventListener("click", function(){
    alert("我是box1的冒泡阶段！");
}, false);
box2.addEventListener("click", function(){
    alert("我是box2的冒泡阶段");
}, false);
box3.addEventListener("click", function(){
    alert("我是box3的冒泡阶段");
}, false);

box1的捕获→box2的捕获→box3的捕获→box3的冒泡→box2的冒泡→box1的冒泡。坑：最内层的box3，先写谁就先执行谁，也就是说对于最内层的box3来说，就不区分冒泡和捕获了。对于不是最内层的其他元素来说，改变事件监听的添加顺序，不会影响执行结果，是先捕获然后冒泡。 DOM2级的事件监听中，如果给同一个元素的同一个事件名的同一个阶段添加多个事件监听，彼此不覆盖，先执行先写的。DOM2事件监听中，this指的是触发事件的这个元素，就是事件传播到的这个元素。

box1.addEventListener("click", function(){
    alert("我是box1的冒泡阶段！A");
}, false);
box1.addEventListener("click", function(){
    alert("我是box1的冒泡阶段！B");
}, false);
box1.addEventListener("click", function(){
    alert("我是box1的冒泡阶段！C");
}, false);
// 弹出顺序A →B→C

低版本IE6、7、8只支持DOM0级，DOM2级是自己发明的attachEvent。所以最最正确工作实践，就是用DOM0级事件监听添加方法，除非基于特别充分的理由或者剋发移动端，否则不需要使用DOM2级。什么时候理由充分必须用DOM2级呢，比如一些新兴事件，如手机事件，touchstart、touchmove、touchend事件等等，只能用DOM2级添加。 事件监听的移出，DOM0级事件监听的移除很简单，直接把onclick属性设置为null即可：box.onclick = null;DOM2级事件监听移除的时候，必须指名道姓去移除。也就是说，如果是匿名函数添加的事件监听，是不可能去掉的:box.removeEventListener("click",boxClickHandler, true)。

# 事件对象event、阻止冒泡、阻止默认事件

在任何事件中，都有事件对象。系统会往我们的事件处理函数中，传入一个实参，就是事件对象，里面封装了你这次点击（或者鼠标进入、键盘按下）的细节，比如鼠标位置、是否按下了alt键等等。我们一般用变量event来接受，注意，不是必须的，可以起别的名字。

box.onclick = function(event){
    console.log(event);
}

兼容性问题，IE6、7、8中，event这个事件对象不是实参！而是全局变量！系统会在每一个事件发生的时候，都去更改这个全局变量的值!

//兼容性写法
box.onclick = function(event){
    //因为在Chrome中，event是实参！在IE6、7、8中event是全局变量！所以我们用或者的短路语法，让它支持哪个用哪个。
    var event = event || window.event;
}

通用属性（注意兼容），bubbles属性，表示事件是否冒泡。基本上所有事件这个属性都是true，表示这个事件冒泡。但是一定要记住：onmouseover鼠标进入、onmouseenter 鼠标进入事件不冒泡。当一个内部元素鼠标进入了，会这个这个元素的onmouseenter事件处理函数，但是事件不会继续往外层传播。它的外层盒子是不会触发onmouseenter事件的，因为事件不冒泡！onmouseenter/onmouseleave事件比类似的onmouseover/onmouseout事件好用，后者冒泡。除了onmouseenter/onmouseleave之外，没有其他重要的事件不冒泡了。

target、srcElement属性，表示事件发生的最内层元素，chrome同时支持上面三个属性，IE只支持srcElement：

var target = event.target || event.srcElement;

用这个属性可以优化代码效率，可以制作''事件委托"。把子元素的事件，委托给父亲，父亲通过event.target来判断是谁触发了事件。不使用事件委托，我们将会有很多事件监听；使用事件委托，事件监听只有1个。提升了页面的效率。比如，小圆点的监听，不要给所有的li了，直接给ol。让ol的event.target这个元素有cur，其余没有cur。

$("ol").mouseover(function(event){
    //event的兼容
    var event = event || window.event;
    //得到事件最小的元素
    var target = event.target || event.srcElement;
    //转为jQuery对象，让触发事件的最小元素有cur，其他元素没有cur
    $(event.target).addClass("cur").siblings().removeClass("cur");
});

当页面上有大量的重复元素需要添加监听的时候，比如100个、200个，最好使用事件委托。 阻止冒泡（注意兼容，冒泡阶段），在冒泡阶段，阻止事件继续冒泡：

event.stopPropagation()；
// IE使用：
event.cancelBubble = true;
// 兼容写法：
if(event.stopPropagation){
    event.stopPropagation(); //高级
}else{
    event.cancelBubble = true; //IE6、7、8
}
// 阻止默认事件：
event.preventDefault();
// IE6、7、8不兼容，
event.returnValue = true;
// 兼容写法：
if(event.preventDefault){
    event.preventDefault();
}else{
    event.returnValue = true;
}

# jQuery中的DOM节点操作

jQuery是当前DOM操作的王者，jQuery创建对象使用$( )函数即可，创建完成的同时返回了一个jQuery，就是这个DOM元素。 8种上树的方法：

// 插入父节点最后面
$("<p></p>").appendTo("#box");
$("#box").append($("<p></p>"));
// 插入父节点最前面：
$("<p>新</p>").prependTo("#box");
$("#box").prepend($("<p>新</p>"));
// 创建一个紧挨后面的新兄弟:
$("#box").after($("<div>新的盒子</div>"));
$("<div>新的盒子</div>").insertAfter("#box");
// 创建一个紧挨前面的新兄弟:
$("<div>新的盒子</div>").insertBefore("#box");
$("#box").before($("<div>新的盒子</div>"));

节点删除自己：.remove()；节点清空内部元素：.empty()。

被包裹：'$("span").wrap("<div></div>");（所有span放在一个盒子里）；一个一个被包裹：$("span").wrapAll("<div></div>")。

克隆节点，不需要写true，jQuery默认就是深度克隆，克隆内部所有节点。如果写true表示将事件监听一并复制。复制的那个对象的事件处理函数，也一并成为了新的元素的事件处理函数。
