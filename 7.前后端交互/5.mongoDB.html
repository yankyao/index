# 5.MongoDB

# NoSQL数据库

传统的数据库MySQL、SQL Server、Oracle、Access都是SQL结构型数据库。他们这些结构型数据库的特点，就是''表"有明确的''字段"的概念，所谓的字段就是''列"：

  • 结构型数据库的缺点：如果今后需要增加字段，之前的所有的条目，也要一起增加。比如你的条目已经有2万条，此时如果突然要增加一个语文成绩字段，此时20000条已经存在的条目都要进行字段的改变，此时特别耗费时间。
  • 结构型数据库的优点：查找快、有主键的概念、从键、主从查找、映射等等的高级数据库的概念。

结构型数据库已经不能完全满足当今社会的需求。 对于一些简单的数据API，只需要保存即可，并不需要非常精确的多个表之间查找然后拼装数据返回给使用者。所以诞生了NoSQL，非关系型数据库。数据没有行、列的概念，使用k-v对儿或者JSON进行存储。优缺点正好和SQL型反过来了，适合结构变化，不适合精确查找。刚好适应了时代''只要存储，不要查找"。NoSQL有一个昵称，叫做Not Only SQL，不仅仅是数据库。实际上功能比SQL弱太多。

两种工作中最常用的NoSQL数据库：MongoDB(文档型数据库)、Redis(K-V对型数据库)，他们都是。NoSQL数据库现在已经广泛的应用于：社交平台、APP通信记录、银行流水、电信通话详单等等。任何只需要保存，但是不需要精确查找的使用场景，都可以用非关系型数据库。
