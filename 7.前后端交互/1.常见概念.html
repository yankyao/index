# 1.常见概念

# Hash

#号的英文就是Hash； 在Ajax技术有一个问题，就是URL是不变的，这样不便于分享网址，此时用Hash解决。

// JS中得到Hash和更改Hash：
window.location.hash
window.location.hash = '锚点名字'

锚点的本质是a标签，a标签可以发出超级链接，也可以定位超级链接：

<!-- 方法1 name/id -->
<a name="add">one...</a>
<a id="add">one...</a>

<a href="#one">跳到one</a>

URL上会添加#one，页面滚动到one的顶部位置。

# JSONP跨域

浏览器安全限制(同源策略)，不允许浏览器Ajax请求自动跨域。 页面在127.0.0.1上就使用Ajax自动访问127.0.0.2。

需要跨域才能访问的资源：

  • 1.后端请求到数据发到本域前端，或者通过Nignx进行代理;
  • 2.JSONP跨域；
  • 3.利用script/img标签请求资源不受同源限制的特点进行jsonp跨域访问；

JSONP就是JSON with Padding ( JSON和 赘语），指的就是把函数的执行放到外部文件，HTML用script标签引用这个js文件，实际上引用的是对一个函数的执行。此时函数定义在HTML文件里面，数据就通过函数实参、形参的使用就进入了js处理的线程。

<!-- JSONP的缺点：
① 太不安全，JSONP的提供者，一定要可靠！
② 没有明显的异步的特点，没有提供回调函数。
③ 需要动态的创建一个script标签，设置了src，一旦上树，上行请求将发出，此时函数将执行： -->
<script type="text/javascript">
    function request(URL, callback){
        //你第二个传入的参数就是哈哈函数
        window.fun= callback;
        var oscript = document.createElement("script");
        oscript.src = URL;
        document.head.appendChild(oscript);
        document.head.removeChild(oscript);
    }
    request("/a.txt",function(data){
        alert(data.data);
    });
</script>

JSONP和Ajax都是异步的； JSON不是Ajax技术，因为没有用到XHR(XMLHttpRequest);

TIP

Ajax技术，Asynchronous Javascript and XML。异步Javascript和XML，让客户端异步访问服务器的一整套技术。 主要是使用XMLHttpRequest，js中的内置构造函数来实现。

# MIME

MIME类型，就是MIME(Multipurpose Internet Mail Extensions)多用途互联网邮件扩展类型。在HTTP中，这个类型写在下行响应的Content-Type中，浏览器依靠这个Content-Type来识别这个东西是一个图片还是一个音乐文件，还是一个css，还是一个html。而不是通过URL的拓展名。比如：http://127.0.0.1:3800/shuaige.mp3路由到一张jpeg图片上，但是这个地址的Content-Type必须是images/jpeg。
