# 3.canvas

# 关于canvas

  H5非常重要的更新，是一对双标签；行内元素、图形容器；集成在浏览器里；宽、高写成行内式，否则会扭曲变形；标签里可以写文字，在不支持canvas的浏览器上显示文字；

  使用步骤：通过ID获取canvas；获取canvas的上下文，打点getContext("2d”)。

<canvas width='800' height='600' id='myCanvas'>对不起，您的浏览器不支持canvas画布，请升级浏览器</canvas>
<script>
    // 得到canvas
    var myCanvas = document.querySelectorAll('#myCanvas')
    // 在画布的context中绘制
    var ctx = myCanvas.getContext('2d')
    ctx.fillStyle = 'lightseagreen'
    ctx.fillRect(100, 200, 100, 200)
</script>

# 笔触、填充

  笔触也叫作“描边”，Canvas中的任何形状都是由笔触、填充两个部分组成的。笔触在canvas中视为一个“Path”的实例，描边用stroke；填充用fill才能上屏幕。

// 笔触相关API
// 开始绘制路径
ctx.beginPath()
// 自动封闭路径
ctx.closePath()
// 将画笔移动到某个点
ctx.moveTo(100, 200)
// 用画笔画线，此时的参数是终点位置，也是下次的起点
ctx.lineTo(200, 200)
// 画线
ctx.stroke()

// 描边：
// 画线之前设置线的宽度和颜色
ctx.lineWidth = 10
ctx.strokeStyle = 'red'
ctx.stroke()

// 填充：
// 设置填充颜色
ctx.fillStyle = 'red'
// 绘制一个填充矩形
ctx.fillRect(100, 100, 300, 200)
// 填充
ctx.fill()

  fillRect()是一个快捷方法，让你省略了beginPath、moveTo、lineTo。所以fillRect(100,100,300,200)等价于：

ctx.moveTo(100, 100)
ctx.lineTo(400, 100)
ctx.lineTo(400, 300)
ctx.lineTo(100, 300)
ctx.closePath()
ctx.fill()

# 弧和圆形

  画弧线，使用的是arc方法，属于path。所以在画弧之前，要beginpath。

var myCanvas = document.querySelectorAll('#myCanvas')
var ctx = myCanvas.getContext('2d')
// 开始绘制
ctx.beginPath()
// 圆心坐标200，200；半径100；0是开始角度；1是终止角度；true表示逆时针
ctx.arc(200, 200, 100, 0, 1, true)

  注意：在canvas中的弧度的角度使用的都是弧度制，1弧度约等于57.3角度；完整的圆终止角度就是Mathi.PI * 2。线型、虚线的绘制，可以参考MDN学习。

# 文字

  fillText函数用来写字，参数是：文字内容、坐标位置；可以使用font属性来调整字号和字形：

ctx.fillText('文字内容', 100, 100)
ctx.font = '50px 微软雅黑'
ctx.fillText('你好', 100, 100)

# 使用图片

  使用图片有一个固定的语法，必须new Image()，然后设置src，设置img.src就会给服务器发送请求，监听load事件。使用ctx.drawImage()函数上画布：

var img = new Image()
img.src = 'images/1.jpg'
img.onload = function() {
    // ctx.drawImage()里面的参数是图片对象、左上角的坐标。可以有四个数字参数，增加了宽度、高度的设置，此时图片会被扭曲：
    // 使用切片，就是9个参数：ctx.drawImage(img, 168, 150, 126, 141,100, 100, 250, 80)
    // 分别表示切片的x,y,width,height,在屏幕上的x,y,width,height
    ctx.drawImage(img, 100, 100, 250, 80)
}

# 运动

  canvas中元素不能运动的，因为上屏幕之后就再也得不到它了，没有任何变量能够持有一个。所以，canvas中运动的原理就是：不断的清屏、重绘，利用视觉残留，形成运动动画。清屏➡️更新➡️渲染➡️清屏➡️更新➡️渲染➡️...

// 清除canvas中的矩形区域，一般是清除整个canvas画布
ctx.clearRect(0, 0, 800, 600)

  使用面向对象在canvas中制作运动的基本结构：

// 类文件
function Circle() {
}
Circle.prototype.render = function() {
}
Circle.prototype.update = function() {
}
// 入口文件
var circle = new Circle()
setInterval(function() {
    ctx.clearRect(0, 0, 800, 600)
    circle.update()
    circle.render()
}, 20)

  每一帧都要更新、渲染。必须提供update、render方法。 但是在实际项目中，演员一般不止一个，但是都要在一个舞台上表演：面向接口编程，接口就是具有一定方法的类的集合。也就是每个类都有render、update方法，此时Actor（演员）就是一个接口，Circle和Fang类都是这个接口的实现。“接口就是类的类”。Java中有明确的Interface(接口)的概念，还有implement实现接口的关键字，当然这是Java的概念。JS中没有明确的接口的关键字，但是可以用继承表示这个“接口”的概念。所以，我们要制作Actor类，Actor类提供update、render方法，并且会把自己放入总数组中。所有演员类，都必须继承这个类，重写update和render方法。

# canvas变形

  在canvas中，所有的变形都是针对canvas的上下文的，而不是某一个要绘制的元素。canvas中进行变形有一套方法：

ctx.save()
ctx.translate(x,y) // 移动坐标系原点
ctx.restore()

ctx.save()
ctx.rotate() // 旋转坐标系
ctx.restore()

ctx.globalCompositeOperation = "type" // 是字符串加引号

# 像素操作

  到目前为止，我们尚未深入了解Canvas画布真实像素的原理，事实上，你可以直接通过ImageData对象操纵像素数据，直接读取或将数据数组写入该对象中。控制图像使其平滑（反锯齿）以及如何从Canvas画布中保存图像。得到像素：

// 其中,0,0表示起点位置，560是宽度，300是高度。
var myImageData = ctx.getImageData(0,0,560,300);

  得到的值可以被使用作为查看初始像素数据。每个像素用4个1bytes值(按照红，绿，蓝和透明值的顺序; 这就是"RGBA"格式) 来代表。每个颜色值部份用0至255来代表。每个部份被分配到一个在数组内连续的索引，左上角像素的红色部份在数组的索引0位置。像素从左到右被处理，然后往下，遍历整个数组。然后,得到5603004 = 672000长度的数组。

# 碰撞检测

  游戏中的碰撞检测一般使用的是AABB盒，即轴对称包围盒,Axis Aligned Bounding Box。在flappybird中，鸟是一个的形状不规则，碰撞的时候取一个大概的矩形盒子来进行判断:

// 鸟这个图片本身是48宽度，碰撞盒是34，所以两边有误差7。
this.A = this.y + 7
this.B = this.x + 41
this.C = this.y + 41
this.D = this.x + 7
// ABCD是边的x或者y，横边就取y值，纵边就取x值。
// 碰撞检测：
if(
    game.bird.A < this.C1 && game.bird.B > this.D1 && game.bird.D < this.B1
    ||
    game.bird.C > this.A2 && game.bird.B > this.D2 && game.bird.D < this.B2
){
    console.log("撞！");
    clearInterval(game.timer);
}

# 事件监听

  canvas上必须检测范围，因为得不到元素。点击鼠标的时候，要遍历每个元素所占的范围，看点没点击到这个元素，如果点击到就存入变量。然后让它的的元素x、y跟随你的鼠标变化即可。

# 场景管理类

  Scene就是场景，FlappyBird就是三个场景：开始画面、游戏过程、结束画面。场景现在开始接手管理自己的演员，而不是game类。但是，场景还是应该把演员注册到game身上，但是管理的权限、谁上台、谁下台，是场景管理器的任务。监听都是场景管理器负责注册！

# requestAnimationFrame

  HTML5中增加了一种定时器，requestAnimationFrame() 请求连续动画帧。这个函数不能设置间隔事件，系统会自动调整最优秀的间隔事件，让你的电脑又不卡，又流畅，又性能又好。和之setInterval有很大的不同，这个定时器表示自动寻找一个合理的时间间隔，调用函数执行一次。所以它像setTimeout()而不是类似setInterval()。

// 开始游戏
class Main {
    constructor(props) {

    }
    start() {
        // 调用主循环
        this.mianLoop()
    }
    mianLoop() {
        console.log(Math.random())
        window.requestAnimationFrame(arguments.callee)
    }
}

  但是requestAnimateFrame不会让你写任何的数字，也就是说不会让你设置任何的帧率。计算机会根据自己的硬件好坏，自动调整帧率！

  fps对游戏有用，比如你的管子120帧出现一个，那么电脑好的用户，实际上变相增加了它的难度。电脑好的用户，可能1秒就有150帧，不到1秒1个管子了。电脑差的用户，电脑自动调整为1秒20帧，此时6秒才会有一个管子。所以不合理！！所以我们你必须要计算fps。frames per seconeds，每秒多少帧。如何计算一秒多少帧呢？很简单，打时间戳！看看每一个时间戳的间隔，和最初时间戳的间隔，有没有大于1000.大于1000了，此时间隔的帧数就是fps。
