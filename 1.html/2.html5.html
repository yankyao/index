# 2.html5

# 概述

  2008年发布。HTML5不仅仅是超文本标记语言，而是一套完整的浏览器新API的综合；新的语义标签、新的本地存储、设备兼容特性（移动端）、连接特性、新的音频视频、新的画布和svg、性能与集成特性、新的样式表；HTML5不是HTML的新版本，而是包含HTML、CSS、JavaScript、浏览器的一整套API。

  兼容情况：移动端兼容较好，PC端慎用；兼容性查询网站：http://html5test.com/ 和 http://caniuse.com/

  新的骨架，新语义标签，新的语言规范；参考：https://developer.mozilla.org/zh-CN/docs/Web/Guide/HTML/HTML5

# 新语义标签

  HTML5推出了新的大纲标签，他们都是双标签，都是div性质，块级元素，是容器标签。

<header>  头部语义
<footer>  尾部语义
<article> 文章语义
<section> 区域语义
<nav>     导航语义
<aside>   侧边内容语义
<main>    主要内容语义

# 新表单元素

  以前的表单元素type类型：

* button
* checkbox
* file
* hidden
* image
* password
* radio
* reset
* submit
* text

  新的表单元素：

<input type="number" min="2" max=“140” />
<input type=“date” />
<input type=“week” />
<input type=“color” />
<input type=“time” />
<input id="range" type="range" min="60" max=“250” />
<input type=“search” />
<input type=“tel" />
<input type=“url" />

  新的表单模式：控件不一定写在form里面了，可以给form加上id属性，然后控件用form属性表示自己是哪个表单里面的控件；required 表示必填项目；autofouse自动获得焦点；placeholder占位符；readonly只读；pattern正则表达式过滤；

<input type="number" min="2" max=“140” placeholder=‘请输入2-140之间的数字’/>
<input type=’text’ required pattern=“^[\d]{2,5}$"/>

  新的控件，label简化写法，直接包裹即可

<input type=’text’ list=’tip’/>
<datalist id=’tip’>
    <option value=‘HTML’>超文本标记语言</option>
    <option value=‘JS’>JS脚本语言</option>
    <option value=‘CSS’>层叠式样式表</option>
</datalist>
<button>按钮</button>
<label><input type=“checkbox”/>足球</label>

  新的表单事件：oninput，前端页面开发的很多情况下都需要实时监听文本框输入，比如腾讯微博编写140字的微博时输入框hu9i动态显示还可以输入的字数。过去一般都使用onchange/onkeyup/onkeypress/onkeydown实现，但是这存在着一些不好的用户体验。比如onchange事件只在键盘或者鼠标操作改变对象属性，且失去焦点时触发，脚本触发无效；而onkeydown/onkeypress/onkeyup在处理复制、粘贴、拖拽、长按键（按住键盘不放）等细节上并不完善。

  onpropertychange属性可在某些情况下解决上面存在的问题，不用考虑是否失去焦点，不管js操作还是键盘鼠标手动操作，只要HTML元素属性发生改变即可立即捕获到。遗憾的是，onpropertychange为IE专属的。其他浏览器下如果想要实现这一实时监听的需求，就要用到HTML5中的标准事件oninput，不过IE9以下的浏览器是不支持oninput事件的。所以我们需要综合oninput和onpropertychange二者来实现文本区域实时监听的功能。

# 新的全局属性

  contenteditable，模拟输入框。但是复制到模拟输入框元素的文本会源保留格式，input则不会保留格式。

<style>
    .input{
        width:200px;
        height:24px;
        line-height:24px;
        font-size:14px;
        padding:5px 8px;
        border:1px solid #ddd;
        border-radius: 12px;
    }
    .input:empty::before {
        // 模拟placeholder，attr(placeholder)定义显示在该选择器之前或之后的选择器的选择的dom元素的某属性。
        content: attr(placeholder);
    }
</style>

<body>
    <div class="input" contenteditable placeholder="请输入文字"></div>
    <script>
        var input = document.getElementsByClassName('input')[0]
        input.oninput = function(event) {
            console.log(event.target.innerHTML)
        }
    </script>
</body>

# 视频和音频

  不用H5插入视频的步骤：将视频上传到视频网站上（优酷、土豆等等），这样就不用买视频服务器。通过分享得到引用链接，使用Flash播放。

<embed src=‘https://player.youku.com/…/…**.swf' quality=‘high’ width=‘100px’ height=‘100px’ align=‘middle’ allowScriptAccess=‘always’ allowFullScreen=’true’ mode=’transparent’ type=‘application/x-shockwave-flash’> </embed>

  H5中引入了video标签，src资源地址，autoplay自动播放，controls显示控制按钮，loop循环播放。视频的方法有pause()、play()：

<video src=‘video/lcw.mp4’ width=‘100px’ height=‘100px’ autoplay controls loop></video>

  音频audio标签，使用方法与video相同。

# 本地存储

  Cookie与LocalStorage、SessionStorage，长时间以来Cookie都是前端开发能够在客户电脑中持久数据的途径：服务器在响应头（response Header）中Set-Cookie，实际上是在客户端浏览器中存储了Cookie小文件；前端开发用document.cookie来读取，操作；但是，Cookie本身不是用来持久数据的，我们之前学习的MySQL，数据库是在服务器上，数据被持久在了服务器上，而不是本地。

  H5提供了新的API，让我们可以在客户端的电脑中，存储数据。LocalStorage（本地存储）、SessionStorage（会话存储），统称为本地存储；提供了很简单的API：

localStorage.setItem(k, v)
localStorage.getItem(k)

  本地存储数据库会自动的为每一个网站（ip地址）建立一个独立的表格，在同一个网站（ip地址）下数据是可以共享的。但是不能跨域（也不能跨浏览器）。localStorage存储的东西，是永远不会丢失的，关机、重启、注销都不会导致数据丢失。除非你清除了记录。而且，本地存储没有字节限制。

  sessionStorage就不一样，它是浏览器窗口一旦关闭，数据丢失了。

  localStorage不能跨浏览器存储，每个浏览器都有自己的小数据库。Chrome存储的，火狐看不见。localStorage是一个非常简单的数据库，没有查询功能、没有SELECT语句，不能用SQL操作，只能简单的存储、读取kv对。和document.cookie类似，V的类型也是字符串；用JSON来持久数据，H5的本地存储API能存储的是K V对，K相当于一个编号，V才是存储的数据，在应用开发中，数据的通用形式是JSON，所以在V中存储JSON形式字符串是主要的目的；JSON的一些操作（原生JS）：

JSON.stringify() // JSON转为字符串

  表单序列化和表单复原，表单复原可以用原生（利用name和value的关系提取为数组，再用“&”join）；也可以直接用jQuery的$("form").serialize()（表单写在form标签里）。表单序列化，则要考虑到转码的问题。

$(document).ready(function(){
    var pre = localStorage.getItem(‘hhhh.cn’)
    if(pre) {
        var preArr = pre.split(‘&’)
        for(var i = 0, len = preArr.length; i < len; i++){
            var itemArr = preArr[i].split(‘=‘)
            var k = itemArr[0]
            var v = itemArr[1]
            switch($('[name=' + k + ‘]’)[0].type) {
                case ’text’:
                case ‘password’:
                case ’number’:
                case ‘url’:
                    $(‘input[name=‘ + k + ‘]’).val(decodeURIComponent(v));
                    break;
                case ‘radio’:
                case ‘checkbox’:
                    $(‘input[name=‘ + k + ‘]’).each(function(){
                        if($(this).attr(‘value’) === decodeURIComponent(v)) {
                            $(this).attr(‘checked’, true);
                        }
                    })
                    break;
                case ’select-one’:
                    $(‘[name=‘ + k + ‘] option’).each(function(){
                        if($(this).attr(‘value’) === decodeURIComponent(v)) {
                            $(this).attr(‘checked’, true);
                        }
                    })
                    break;
            }
        }
    }
})

# websocket

WebSocket protocol 是HTML5一种新的协议。它实现了浏览器与服务器全双工通信(full-duple)。一开始的握手需要借助HTTP请求完成。

  • HTTP设计的时候，比如浏览器直接发出请求，服务器才能响应。如果浏览器不发出request请求，服务器是不能主动找到浏览器，传输一些数据的。也就是说：浏览器必须主动请求，服务器才会发出响应。

  但是现在的web页面，对实时的要求很高：web看股票、篮球比赛图文直播、聊天室、站内信等等。现在的做法基本都是长轮询，用通俗易懂的话来说，就是客户端定时（每隔大约1秒钟）向服务器发送请求以获取最新的数据信息。断开、连接、断开、连接。服务器是不能有新消息就主动通知浏览器的。

Header，互相沟通的Header是很小的-大概只有 2 Bytes；
Server Push，服务器的推送，服务器不再被动的接收到浏览器的request之后才返回数据，而是在有新数据时就主动推送给浏览器。

websocket需要浏览器足够新，IE10+。服务器也要足够新，NodeJS天生就支持。

socket.io可以提高开发websocket的效率，已经封装好了诸如HTTP头等
