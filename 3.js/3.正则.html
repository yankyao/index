# 3.正则

# 基础

  用途，regular expression，简写为RegExp；用来匹配字符串中的字符组合的模式，常用来做表单验证、字符串数据的修改等；在JS中，正则表达式也是对象类型数据，是一种索引类型，object对象型数据。   正则表达式的字面量，语法为 /正则语句/ ；一般把正则表达式传给一个变量。   常用方法，谁的方法谁调用：

// 字符串方法
split()       // 根据匹配字符串，来切割父字符串
match()         // 使用正则表达式与字符串相比较，返回一个从字符串中匹配正则的结果的数组
search()        // 对正则表达式规定的指定字符串进行搜索，返回父字符串中出现的第一个匹配项的下标
replace()       // 用正则表达式和字符串相比较，然后用新的字符串来替换父字符串中匹配的部分字符串
// 正则方法
exec()         // 在目标字符串中执行一次正则匹配操作
test()          // 测试当前字符串能否匹配正则表达式

# 正则的术语和操作符

正则表达式由普通字符和特殊字符（又叫元字符）组成。

普通字符包括大小写的英文字母、数字，元字符则是具有特殊意义的符号；正则中的元字符包括：（）[ ] ｛｝\ ^ $ | ? * + 。在表达式中要匹配字符串中的出现的元字符，要用到转义字符\。

精确匹配，在//中间输入要匹配的固定值或者常量（没有特殊字符），里面的字符顺序不能颠倒，排序在父字符串中紧挨； 预定义特殊字符串:

\t       // 制表符      
\n、\r   // 回车符或者换行符  
\f       // 换页符 
\b       // 空格

字符集，正则表达式中，用一类字符串集合去匹配字符串中的一个字符。字符集用[ ]包裹：

  • 简单类：在[ ]中罗列出来，不需要任何其他符号，直接连写；
  • 范围类：[0-9]、[a-z]、[A-Z]；
  • 组合类：[0-9a-z]、[0-9a-b]、[1-6H-Y]；
  • 负向^：表示不是中括号里的，例如：[^e-l]；

  修饰符，写在/ /后面。g修饰符用于执行全局匹配，查找所有匹配而不是在找到第一个后停止；用于match和replace方法中；i修饰符用于执行对大小写不敏感的匹配。两个修饰符可以同时使用，直接连写即可。   边界，开头^、结尾$：/^ $/。\b用于查找位于单词（连续数字或者字母）的开头或者结尾的匹配；\B用于查找不处在单词（连续数字或者字母） 的开头或者结尾的匹配（连续数字或者字母中间）；对比记忆：单词边界\b必须包括单词边界。非单词边界\B不能有单词边界。

  预定义类：

[^\n\r]    // 除了回车和换行之外的任意字符；
\d [0-9]   // 数字字符
\D         // 非数字字符
\s         // 空白字符  空格 缩进 换行 换页
\S         // 非空白字符
\w         // 单词字符[0-9a-zA-Z_]数字大小写字母下划线
\W         // 非单词字符

  量词，正则表达式中，量词规定的数量的作用范围，是量词前面的一个字符或者分组。

｛n｝       // n可以是任何正整数和0；表示出现的次数；
｛n，m｝    // 表示连续出现（紧挨，不能有空格或其他字符）至少n次，最多出现m次；
｛n，｝     // 至少出现n次；
？         // 出现0次或者1次；
*          // 出现任意次；
+          // 出现1次或多次，至少出现1次；

  分组，量词能处理紧密相连的单个字符或者同类型字符；小括号对字符进行分组，量词的作用范围是小括号里的所有字符。

"abcdef".match(/(ab|cd)+|ef/)   // ["abcd", "cd", index: 0, input: "abcdef”]

  或操作符|，符号前面的或者后面的；利用小括号分组，指的是是小括号里的或关系。   分组的反向引用，反向引用标识是对正则表达式中的匹配组捕获的子字符串进行编号，通过“\编号(在表达式中)”，或者“$编号（在表达式外）”进行引用。从1开始计数。

// 在正则表达式里反向引用：
/(\w{3})/.test('byebye’)     // true
/(\w{3})\1/.test('bye’)      // false
/(\w{3})\1/.test('byebye’)   // true
/(\w{3})\1/.test('byebyy’)   // false

// 在正则表达式外:
'abc=def'.replace(/(\w{3})=(\w{3})/, '$2=$1’)    // "def=abc”
'123*456'.replace(/(\d{3})/g, function(a,b,c){console.log(a,b,c)})    //函数内输出两次，第一次123 123 0，第二次456 456 4。 最终返回"undefined*undefined”

'123*456'.replace(/(\d{3})\*(\d{3})/g, function(matchPart,$1,$2){
    console.log(match,$1,$2); 
    return $2-$1
})                                                                    
// 函数内输出一次123*456 123 456。最终返回"333"
// replace相当于把字符串中符合正则的各个部分取出来；function对字符串中匹配到的各个部分进行重组

  中文匹配，固定用法[\u4e00-\u9fa5]。   正向肯定断言?=，非获取匹配？：

function groupByCommas(num) {
    return num.toString().replace(/\B(?=(\d{3})+(?!\d))/g, ',')
}
groupByCommas(3453453453453)   // "3,453,453,453,453"

a-zA-Z和A-z的区别；A-z包含_等符号。
